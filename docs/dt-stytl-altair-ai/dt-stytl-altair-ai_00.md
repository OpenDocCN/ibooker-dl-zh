# 第一部分：介绍 Altair 和生成式 AI 到数据叙事

什么是数据叙事？你如何使用 Python Altair 实现数据驱动的叙事？生成式 AI 会为构建数据故事带来哪些好处？你将在本书的第一部分找到这些问题的答案。这一部分以渐进的方式介绍生成式 AI 在数据叙事中的应用：首先，我们将探讨 GitHub Copilot，然后在第四章，我们将讨论 ChatGPT 和 DALL-E。我选择遵循这种学习策略，因为先建立理论基础以理解主要概念，然后再利用生成式 AI 提供的各种工具进行自动化，这样做会更好。采用这种方法，你将成为生成式 AI 工具的全能大师，而不是它们的仆人。

在我们进入结合生成式 AI 的奇妙数据叙事世界之前，我想提醒你一件事：生成式 AI 是一个不断发展的领域，所以你在本书中读到的代码在你尝试运行它的时候可能已经过时了（尽管在写作时已经更新到最新版本）。然而，描述的原则始终有效，你可以查看生成式 AI 工具的官方文档来更新代码。实际上，本书中描述的代码位于一个 GitHub 仓库中，因此你可能甚至可以考虑在损坏的代码上提出问题，以保持 GitHub 仓库持续更新。这将是一种极好的合作方式，如果你这样做，我会非常感激。

在第一章中，你将了解数据叙事是什么，以及为什么你应该使用它来传达数据探索和分析中提取的见解。为了构建数据故事，你还将被介绍到数据、信息、知识、智慧（DIKW）金字塔，这是你将在整本书中使用的模型。尽管存在其他构建数据故事的模型（例如，叙事弧），但我选择 DIKW 金字塔作为本书的参考，因为我相信它更直接、更有效。它就像套娃一样，最外层的部分包含所有之前的部分，尽管每个单独的部分都可以作为一个独立对象存在。就像套娃（也称为嵌套娃娃）一样，DIKW 金字塔的每一步都可以作为一个独立（部分）故事存在；然而，只有最后一步，智慧，包含了一个完整的故事。在本章的最后部分，你将看到一个实际的应用案例，它应用了 DIKW 金字塔，这将揭示这个模型的可能性。我希望这个简单的例子能激发你的兴趣。然而，如果它没有达到你的期望，我请你耐心等待。在整个书中，你将遇到许多其他实际案例研究和例子，我希望你能将它们作为你自己在现实世界场景中的参考。

在第二章中，你将通过使用 Altair 和生成式 AI 编写你的第一个数据故事来亲自动手。目前，你将仅使用 Copilot，但请耐心等待。你将在本书的后面部分学习如何使用 ChatGPT 和 DALL-E。在这一章中，你将只看到 Copilot 在工作。你不仅将实现示例，而且将学习如何将 Copilot 作为工作工具使用，不仅限于 Altair，而且可以生成任何类型的代码。

第三章将回顾 Altair 的基本概念，以及 Altair 背后的数据可视化语法 Vega 和 Vega-Lite。你将实施实际练习和一个最终案例研究，使用 DIKW 金字塔。与之前章节中实现的案例研究相比，这个案例稍微复杂一些。在这一章的结尾，你将拥有足够的 Altair 技能，准备好使用 ChatGPT 和 DALL-E。

在第四章中，你将学习如何为 ChatGPT 和 DALL-E 构建提示，以及如何使用这些工具进行数据故事讲述。在使用它们之前，你将回顾一些与人工智能、机器学习、深度学习和生成式 AI 相关的一般概念。这将帮助你设定生成式 AI 工具的背景。在章节的最后部分，你将实施一个实际用例，展示生成式 AI 在数据故事讲述中的潜力。

如下列表所示，在每一章的结尾，你将实施一个实际案例研究，每个案例都有不同的目的：

+   *第一章*—这个案例研究仅是理论性的（没有代码）且关注与宠物相关的*广告活动*的一些统计数据。你可以在本书的 GitHub 仓库中找到这个例子的代码。

+   *第二章*—这个案例研究描述了一个与酒店建造新游泳池机会相关的简化*决策过程*。你将实现两个版本：一个使用 GitHub Copilot，另一个不使用。

+   *第三章*—这个案例研究聚焦于一个类似*数据新闻学*的例子，研究北美的人口增长。这个例子比前几章中实现的例子更高级。

+   *第四章*—这个案例研究再次涉及*决策过程*。这个案例增加了生成式 AI 到先前的案例研究中。
