# 前言

欢迎来到《PyTorch 中的 AI 和 ML 程序员指南》。我的机器学习（ML）之旅始于许多年前，那时我使用的是 Lisp 和 Prolog 等语言和框架。之后，我的旅程把我带到了谷歌，在那里我帮助启动并扩大了 TensorFlow。这次经历为我的上一本书《[程序员 AI 和机器学习](https://learning.oreilly.com/library/view/ai-and-machine/9781492078180/)》提供了信息。

自从那本书出版以来，每次我遇到社区讨论 AI 时，总会有人问一个问题：提问者是否应该投资时间学习 PyTorch。一开始这个问题很奇怪，但随着我听到的越来越多，我开始开始调查。

那种想法让我走到了职业生涯的这个阶段，PyTorch 曾经是我的竞争对手，现在却是我热情拥抱的东西。为什么？因为它在力量和灵活性之间取得了完美的平衡，让研究人员或高级工程师能够突破极限，同时也足够简单，让任何开发者都能上手并开始他们的 ML 之旅。

这本书的目标是为你，作为一名程序员，做好准备——如果你还不完全理解 ML，它也足够易于理解，同时也会让你接触到帮助你深入学习的先进概念。目标是：让你成为一名 ML 和 AI 开发者，而无需博士学位！

我希望你会觉得这本书有用，并且它能让你充满信心地开始这段奇妙而有益的旅程。

# 谁应该读这本书

如果你对 AI 和 ML 感兴趣，并且想要快速入门，构建从数据中学习的模型，这本书就是为你准备的。如果你对开始学习常见的 AI 和 ML 概念——计算机视觉、自然语言处理、序列建模等——并且想看看神经网络如何训练来解决这些领域的问题，我相信你会喜欢这本书。而且如果你已经听说过围绕生成式 AI 的种种喧嚣，我们将卷起袖子探索它是如何通过基于 transformer 和 diffuser 的模型来工作的。

最重要的是，如果你因为认为这个计算机科学领域难以进入而推迟进入，特别是相信你需要翻阅你旧的计算书籍，那么请不要担心：这本书采用了一种以代码为先的方法，展示了使用 PyTorch 在 ML 和人工智能领域入门是多么容易。

# 为什么我写这本书

我第一次认真涉足人工智能是在 1992 年春天。作为一名刚毕业的物理学士，我住在伦敦，正值一场严重的经济衰退之中，我已经失业了六个月。英国政府启动了一个培训 20 名人工智能技术人员的项目，并向申请人发出了呼吁。我是第一个被选中的参与者。三个月后，该项目惨遭失败，因为尽管可以用人工智能完成大量的理论研究工作，但没有简单的方法将其实际应用。人们可以用名为 Prolog 的语言编写简单的推理，用名为 Lisp 的语言进行列表处理，但并没有清晰的路径将它们部署到工业中。随之而来的是著名的“人工智能寒冬”。

然后，在 2016 年，当我正在谷歌公司负责一个名为 Firebase 的产品时，公司为所有工程师提供了机器学习培训。我和其他一些人坐在一个房间里，听了关于微积分和梯度下降的讲座。我无法将这些与机器学习的实际应用联系起来，我突然回到了 1992 年。我对此提出了反馈，以及我们应该如何教育人们学习机器学习——首先教程序员编写代码。谷歌接受了这一理念，随着 PyTorch 的发布，Meta 也接受了这一理念。

尤其是两者都强调了高级 API，这使得开发者可以轻松入门，我意识到需要一本书来利用这一点，扩大机器学习的普及，使其不再只是数学家或博士的专属。

我认为，更多的人使用这项技术并将其部署到最终用户手中，将导致人工智能和机器学习的爆炸式增长，从而防止另一个“人工智能寒冬”，并将世界变得更好。我已经看到了这种影响，从谷歌在糖尿病视网膜病变方面的工作，到宾夕法尼亚州立大学，再到 PlantVillage 为移动设备构建的机器学习模型，帮助农民诊断木薯病，到无国界医生使用 TensorFlow 模型帮助诊断抗生素耐药性，以及更多更多！

随着生成式人工智能的出现，以及 transformers 和 diffusers 作为独立库的兴起，下一波人工智能的大潮已经到来。PyTorch 是这一切的核心——因此，是时候让我把我的工作更新到最新，并展示你们如何轻松地涉足人工智能和机器学习开发的水域了。

有了这个想法，欢迎来到这本面向 PyTorch 编程人员的 AI 和 ML 入门书籍。我迫不及待地想看看你们能创造出什么。

# 导航本书

本书分为两个主要部分。第一部分（第一章至第十一章）讲述了如何使用 PyTorch 构建适用于各种场景的机器学习模型。它从最基本的原则开始——构建一个只包含一个神经元的神经网络模型——然后通过计算机视觉、自然语言处理和序列建模。第二部分（第十二章至第二十章）将引导你了解生成式 AI 场景——从理解 ChatGPT 等应用中如何使用 Transformer，到 Midjourney 等图像生成中的扩散器。大多数章节都是独立的，因此你可以随时加入学习新知识，或者当然，你也可以从头到尾阅读整本书。

# 需要了解的技术

本书前半部分的目标是帮助你学习如何使用 PyTorch 构建具有各种架构的模型。对此的唯一真正先决条件是理解 Python，特别是 Python 的数据和数组处理表示法。你可能还想要探索 NumPy，这是一个用于数值计算的 Python 库。如果你对这些不熟悉，它们相当容易学习，你可以在学习过程中获取所需的知识（尽管一些数组表示法可能有点难以理解）。
