# 前言

我们正处于软件开发的一个地壳变动时刻。

在过去的几年里，生成式 AI 工具已经从新奇变成了必需品。最初只是 IDE 中的自动完成建议，已经发展成为完整的共飞行员、设计助手和生产级代理，可以调试代码、发布功能或构建整个应用程序。我们构建软件的方式正在改变——而且变化非常快。

作为一名兼职 CTO，在过去十年里，我一直在帮助初创公司更快地交付更好的软件。我的工作是设定开发速度和产品质量的高标准，并确保我的团队能够利用最好的工具和流程。这意味着我需要走在曲线的前面，并尝试最新的工具。我评估哪些工具可以改进我们的工作流程，并将它们整合到我们的开发流程中，并促进其采用。我曾领导过使用 ChatGPT、Cursor、GitHub Copilot 和 Lovable 等工具进行日常软件开发流程的工程团队，这些工具加速了他们的速度并提高了交付质量。

但这令人感到压倒性。每个月都会发布新的模型，每周都会有新的工具推出。而且每天，我都看到开发者以各种不同的方式使用它们：为了更快地编码，为了学习新的技术栈，或者为了在周末从零开始推出产品。我知道我不是唯一一个在努力跟上的人。所以我决定向前迈出一步，测试我能找到的所有工具，并从噪音中提取信号。这本书就是这项工作的结果：为希望明智地利用生成式 AI 的软件工程师提供的一本指南。

# 如何使用这本书

这本书是关于 2025 年软件开发中生成式 AI 状态的实用指南。它不是理论探索或一系列预设教程。相反，我专注于工具比较、实用工作流程和真实案例研究。

我亲自测试并评价了数十种工具，使用相同的提示和挑战。每一章都包括用例、局限性和批判性评估。这不仅仅关乎什么有效，什么无效，还包括如何以及何时使用这些工具，以及如何评估新的工具。

这本书是某个时间点的快照。实际上，有些章节在编辑过程中中期被重写，因为工具景观发生了变化。例如，我第一次撰写关于前端代码生成的文章时，Lovable 还不存在。三个月后，它成为了主导者。所以，在你阅读这本书的时候，可能会有新的工具出现，新的功能将再次重新定义这个领域。

没关系。这本书为你提供了一个评估工具和工作流程的框架，而不仅仅是一份产品评论列表。

# 这本书面向的对象

这本书面向软件工程师、产品构建者、CTO 和希望保持领先的好奇探索者。

无论你是在构建下一个创业公司、领导一个工程团队，还是试图自动化繁琐的工作流程，我的目标是帮助你更清晰地思考生成式 AI 在你的日常工作中能做什么（以及不能做什么）。你将找到实用的策略、关键的评估以及足够的背景知识，以便做出明智的决定，而不需要成为机器学习专家。

# 章节概述

第一章，“代码生成和自动补全” 介绍了软件开发中最广泛采用的 AI 工具类别：代码生成和自动补全助手。它解释了 ChatGPT、GitHub Copilot、Cursor 和 Gemini 等工具如何通过减少模板代码、加速学习和提高生产力来重塑开发者体验。本章包括对 30 多个工具的实际评估，详细介绍了它们的优缺点，并比较了基于浏览器的助手与集成到 IDE 的解决方案。它还介绍了一种在整个书中使用的持续评估方法，根据现实世界的编码挑战对代码生成工具进行评级。

第二章，“用户界面和用户体验设计” 探讨了生成式 AI 如何改变 UI/UX 设计和前端开发。它涵盖了两个主要工具类别：那些从自然语言生成 UI 设计的工具，以及将那些设计转换为可工作的前端代码的工具。通过测试 Uizard 和 Bolt.new 等工具，本章揭示了 AI 如何将设计到代码的工作流程从数周缩短到数小时，使非设计师能够创建界面，并支持开发者使用现成的 HTML/CSS/React 代码。它还讨论了 AI 生成设计的权衡，包括质量、灵活性和原创性，并描绘了这些工具在何处增加价值，以及人类创造力仍然至关重要的地方。

第三章，“错误检测和代码审查” 讨论了人工智能工具如何改变代码审查和错误检测，将它们从缓慢的、仅由人类执行的过程转变为更快、更确定性的系统。它介绍了一系列基于 IDE、集成 Git 和基于浏览器的工具，这些工具通过实时反馈、安全漏洞检测和风格强制自动化代码审查。实际案例展示了这些工具如何捕捉到 SQL 注入、跨站脚本（XSS）漏洞、内存泄漏和低效循环等问题，从而提高初级工程师的质量、速度和教育水平。

第四章，“自动化测试和质量保证”，探讨了生成式 AI 如何简化软件测试和质量保证。它比较了自动化功能性和非功能性测试任务的工具，从从自然语言提示生成测试案例到检测视觉 UI 错误，以及启用自我修复测试。通过现实世界的示例和测试场景，本章突出了 AI 如何减少 QA 的重复性负担，加快发布周期，并使 CI/CD 流程更加可靠，同时仍然强调在边缘情况和判断性决策中需要人类测试员。

第五章，“预测分析和性能优化”，展示了 AI 工具现在如何允许软件开发人员和其他非技术角色使用自然语言提示和简单的用户界面来分析数据集、提取见解并预测未来的行为。这些工具与真实世界的零售数据集进行了测试，评估了它们生成具有商业价值见解、细分用户和产生可靠预测的能力。本章强调了数据智能民主化的承诺和陷阱。

第六章，“文档和技术写作”，探讨了软件工程中最被忽视的部分：文档。它评估了生成内部文档、API 规范、用户指南和变更日志的 AI 工具，一些嵌入在 IDE 或 CI 工作流程中，另一些则是独立的。本章还解释了为什么大多数文档维护得不好（或者根本就没有编写），并展示了 AI 如何帮助更快地生成高质量的文档。每个工具都在真实的项目代码库上进行了测试，并对其深度、准确性、Markdown 格式和可用性进行了评估。

第七章，“聊天机器人和虚拟助手”，探讨了现代大型语言模型如何重新定义了聊天机器人的功能。今天的 AI 聊天机器人已经超越了基于规则的僵化流程，能够进行上下文对话、执行任务并与 API 交互。本章将工具分为无代码构建器、拖放平台和全代码 SDK，评估了每个工具的设置简便性、事实准确性、记忆保持能力和部署准备情况。通过基于零售数据集构建的测试案例，本章展示了 AI 聊天机器人在哪些方面表现出色，在哪些方面存在幻觉，以及构建一个真正有用的助手需要哪些条件。

第八章，“实施成功案例”，最后一章展示了生成式 AI 工具在实际应用中的真实故事，从像 Pieter Levels 这样的独立黑客在几小时内发货游戏，到 Shopify 的企业团队将 Cursor 等工具嵌入到纪律性的开发工作流程中。本章探讨了“氛围编码”的概念，即从编写每一行代码到使用 AI 进行架构设计的转变，以及独立构建者和大型团队在采用动态方面的差异。这些故事说明了所有级别的工程师如何在 AI 增强的未来中重新思考生产力、提示和代码质量。
