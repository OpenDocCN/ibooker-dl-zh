# 前言

人工智能不再是某个遥远、未来的概念。它就在这里。它是真实的。它正在改变我们几乎在所有你能命名的领域的运作方式：软件、营销、医疗保健、金融、教育，等等。曾经是数据科学家和博士的领域已经变得主流。今天，如果你懂得如何有效地使用人工智能，你就有优势。

正是因为这个原因，学习人工智能不仅仅是一项锦上添花的技能——它是一件你需要的东西。当然，有很多种开始学习的方式：在线课程、教程、视频、书籍。但如果你想要脱颖而出，向雇主展示你的认真态度，拥有一个证书可以真正地改变一切。它不仅仅是一张纸。它是证明你知道你所知，并且能够将其应用于重要领域的证据。

如果你正在试图弄清楚哪种证书值得你投入时间，AWS 认证人工智能实践者是一个不错的选择。毕竟，AWS 是从小型初创公司到科技巨头的所有事物的基石。在这里获得认证意味着你不仅在学习人工智能——你还在学习如何使用一些最具创新性公司的工具来构建和扩展智能应用。

考试本身涵盖了所有基本内容：机器学习（ML）、深度学习、自然语言处理（NLP）、计算机视觉、生成式人工智能和负责任的人工智能。但它不仅仅停留在理论上。它将所有内容都与 AWS 生态系统联系起来。你将能够亲身体验这些亚马逊服务：SageMaker、Comprehend、Rekognition、Transcribe 和 Bedrock。

无论你是想要转向新的角色，提升你当前的角色，还是自信地领导人工智能项目，这个认证为你提供了一个坚实的基础。这本书就是为了帮助你达到这个目标。在书中，你会发现实用的建议、易于理解的解释，以及与考试直接相关的丰富指导。

如果你一直在考虑投身于人工智能领域，现在就是时候了。工具触手可及，需求在增长，AWS 为你提供了一个强大的平台来构建。将这本书作为你的发射台。让我们开始吧——你有很多令人兴奋的领域需要覆盖。

# 覆盖内容

下面是对每一章的简要概述：

第一章，“为什么要认证？AWS AI 实践者考试简介”，提供了关于认证的概述。本章详细说明了参加考试的原因，包括职业和薪资的提升，并概述了其目标受众。它还分解了考试的主题、格式、费用以及其他相关的认证考虑事项。

第二章，“AI 实践者的 AWS 基础”，描述了 AWS 平台和云计算的基础知识。本章探讨了不同的云模型（公共、私有、混合）和服务类型（IaaS、PaaS、SaaS）。它还涵盖了 AWS 全球基础设施，包括区域和可用区，并介绍了核心服务如 Amazon EC2、Amazon S3 以及安全共享责任模型。

第三章，“AI 与机器学习”，探讨了关键的 AI 和 ML 概念。它描述了不同类型的 ML，包括监督学习、无监督学习和强化学习。本章详细介绍了完整的 ML 生命周期，从确定业务目标和数据处理到模型部署和监控。它还介绍了 Amazon SageMaker 作为在 AWS 上构建、训练和部署 ML 模型的主要服务。

第四章，“理解生成式 AI”，探讨了生成式 AI 背后的核心技术，如神经网络和深度学习。它详细介绍了各种生成模型，包括 GANs、VAEs、扩散模型和 transformer 模型架构。本章还涵盖了基础模型（FMs），其开发生命周期，微调和检索增强生成（RAG）等优化技术，以及关键评估指标。

第五章，“使用 AWS 工具的 AI 实际应用”，突出了 AI 及其背后的 AWS 工具的实际、真实应用案例。它涵盖了使用 Amazon Rekognition 进行计算机视觉、使用 Amazon Comprehend 和 Amazon Lex 等服务进行 NLP，以及使用 Amazon Textract 进行智能文档处理（IDP）。本章还讨论了使用 AI 进行欺诈检测，并提供了何时使用 AI 解决商业问题的指导。

第六章，“使用 Amazon Bedrock 和 Amazon Q 构建”，深入探讨了 AWS 构建和使用生成式 AI 的主要服务。它详细介绍了 Amazon Bedrock，这是一个通过游乐场、代理和知识库等特性访问和定制 FM 的平台。本章还介绍了 Amazon Q，这是一个适用于商业和开发者用例的 AI 助手。

第七章，“提示工程指南”，提供了如何制作有效的提示来引导 FM 响应的指导。本章将提示分解为四个关键组件：指令、上下文、输入数据和输出指示器。它涵盖了零样本、少样本和思维链提示等最佳实践和技术，同时也讨论了提示注入和越狱等安全风险。

第八章，“负责任 AI 的框架”，侧重于开发和部署 AI 系统所需的原则，以确保其安全、值得信赖和道德。它探讨了关键风险，如毒性、知识产权侵权和偏见，并详细介绍了负责任 AI 的核心原则，包括公平性、可解释性、透明度和安全性。本章还突出了支持这些原则的 AWS 工具，例如 Amazon SageMaker Clarify 和 Amazon Bedrock 的护栏。

第九章，“AI 解决方案的安全、合规和治理”，解释了安全策略和生成式 AI 安全范围矩阵。本章涵盖了关键合规标准，如 HIPAA 和 GDPR，并探讨了数据治理概念，包括数据溯源和保留策略，以及支持这些策略的 AWS 工具。

第十章，“成功通过 AWS 认证 AI 实践者（AIF-C01）考试的策略和技术”，总结了本书中最关键的主题，以指导最终的学习会话。
