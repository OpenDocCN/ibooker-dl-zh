# 附录 C. 探索性测试笔记

这些是第八章中探索的使用案例示例的原始笔记：

**章程**

探索如何在报告视图中渲染预订。

在一个非常大的预订集合中

为了发现大量预订是否易于阅读

**会话笔记**

**会话设置：**

*我如何为报告页面创建测试数据*？

通过提示分析 API 和 JS 中的代码，似乎报告数据是通过从房间 API 拉取数据然后遍历每个房间从预订 API 检索预订来构建的。

**我如何为房间和预订 API 创建 SQL 数据**

1.  创建一个基于`seed.sql`文件生成房间 API 测试数据的提示。

1.  将数据添加到`seed.sql`中，并使用种子数据重新构建房间 API。

1.  创建一个脚本来使用 ChatGPT 生成预订 API 的测试数据。

1.  由于它破坏了测试，因此无法使用包含数据的预订 API 重新构建，所以加载了 IDE 中的预订 API。

**调查**

*当我加载页面时会发生什么*？

+   Bug：当有大量预订时，页面加载数据非常缓慢。

+   页面加载后，我可以在日历视图中看到两个预订，然后有查看特定日“查看更多”的选项。

*当我使用导航控件工作时会发生什么*？

+   Bug：在从繁忙的月份导航到安静的月份时，日历也加载缓慢。

+   尽管页面加载缓慢，但我仍然能够导航。

*如果我想要查看更多预订会发生什么*？

+   Bug：显示当天预订的弹出窗口溢出到页面顶部，意味着某些预订无法阅读。

+   Bug：当有大量预订时，弹出窗口加载缓慢。

*如果我想要创建一个新的预订会发生什么*？

+   仍然可以点击和拖动天来打开管理员预订弹出窗口。

+   Bug：点击取消意味着等待日历组件再次渲染的时间很长。

*键盘可访问性*？

+   能够成功在日历中按 Tab 键。

+   在“查看更多”上按 Enter 键会打开包含更多预订的弹出窗口。

+   Bug：在包含更多预订的弹出窗口中按 Tab 键不友好。我必须先点击，然后才能按 Tab 键，或者在我被带到弹出窗口中的第一个预订之前，必须遍历所有日历元素。

+   Bug：在弹出窗口中按 Tab 键不会将焦点带到溢出到页面顶部的预订上。

**使用 PAOLO 助记符进行调查**

提示想法：报告日历在不同屏幕尺寸和分辨率下以纵向模式显示的方式。

*如果我放大和缩小日历会发生什么*？

+   Bug：调整大小加载缓慢。

+   Bug：放大查看会隐藏“查看更多”按钮，最终隐藏每个日历的整个内容。

+   当缩小查看时，日历渲染正常。

提示想法：测试日历是否能够快速响应用户设备方向的变化，而不会出现延迟。

*如果我尝试不同的屏幕尺寸会发生什么*？

+   Bug：屏幕调整大小很慢，所以在最终自行纠正之前，初始布局很混乱。

+   最终确实正确渲染。

+   很好地处理了方向变化。

提示想法：确保在设备切换到纵向模式时，日历中的文本和标签仍然清晰易读。

+   Bug：在移动屏幕尺寸下，文字非常小且难以阅读。

+   在分辨率更高的移动设备上，如平板电脑，文本更容易阅读。

提示想法：确保在纵向模式下可用的所有功能和功能在横向模式下也可访问且功能正常。

+   Bug：在移动设备视图中无法点击和拖动日历来调用管理员预订。

+   仍然可以在纵向和横向模式下使用导航控件。

+   制表符功能仍然有效。
