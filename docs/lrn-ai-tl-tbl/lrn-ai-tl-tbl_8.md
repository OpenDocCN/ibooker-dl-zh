# Tableau 的未来

自从 Salesforce 收购 Tableau Software 以来，一直有一种观点认为 Salesforce 将消除 Tableau 生态系统中的非云产品。随着 AI 工具的引入，公司几乎已经确定了这一方向。在 Salesforce 年度客户大会 Dreamforce（2024 年 9 月）上，Tableau 总裁兼首席执行官 Ryan Aytay 推出了 Tableau Einstein，这是 Tableau 代码的重写，以便在 Salesforce 的（仅云）核心平台上运行。

Tableau Einstein 是一个重新构想的 Tableau 体验，旨在最终允许组织访问并利用他们已经存储的所有数据。它也被描述为一个完整的分析解决方案，其中数据建模、语义建模、指标创建和可视化都可以在一个严格管理的环境中进行。这一技术的核心是数据云，它提供了零复制连接到市场上所有主要云数据仓库解决方案。与 Tableau Einstein 一起宣布的新概念是“工作空间”的概念，这是一个组织、收集、管理和与适当的受众（内部和外部）共享分析的地方。工作空间可以是任何东西，从个人游乐场到内部团队空间到外部市场。Tableau Einstein 包含了本书中讨论的所有 AI 工具，以略有不同的方式打包。

Tableau Einstein 还为 Tableau 带来了新的用户界面。在 Tableau 中工作，Einstein 与您可能在 Salesforce 的其他产品中熟悉的标签页和组件化资产非常相似。这一概念也呼应了 Tableau 一直倡导的关键信息之一：工作流程中的分析。有了这个新的用户界面、新的 VizQL 数据服务以及一个用于共享和重用分析资产的市场的推出，下一代 Tableau 旨在将可视化数据和基于数据的洞察力编织到用户工作流程的各个方面。尽管仪表板设计仍有空间，但高度关注个性化 contextualization 表明，将在恰当的时间提供小块信息。

## Tableau Pulse

在撰写本书的过程中，我有幸跟踪了 Tableau Pulse 的初始发布以及随后的发布。因此，对我来说，Tableau Pulse 的方向似乎相当明确。已经取得了大量进展，以确保指标更加集成到生态系统中，特别是新宣布的将指标定义包含在数据目录功能中，以及能够在仪表板中嵌入指标对象的能力。

此外，最近还推动添加了新的功能，如为指标设置手动目标、高级排序指标以及作为高级 Tableau+ 功能在指标之间扩展人工智能的覆盖范围。随着 Tableau Pulse 的不断发展，我们可能会看到大多数额外的先进或新功能作为高级功能仅限发布。

Tableau Pulse 仍存在一些较大的差距，我希望能够得到填补。其中最重要的是所有指标都需要与时间维度相关联。虽然时间在分析中非常常用，但在可视化中要求包含时间维度在许多情况下成为限制因素。我希望看到图表类型的扩展，并希望时间维度仅用作过滤器。

我还希望 Pulse 的功能得到扩展，以便不同的指标可以在相同的时间范围和维度过滤器上独立工作——基本上是一个重新构想的仪表板版本，其中相关的指标可以根据全局过滤器进行操作。尽管这在指标的分解部分已经实现，但将功能扩展到调整多个指标以匹配相同的数据过滤切片似乎不可避免，这样可以避免用户混淆并增强提供的总结洞察。同样，开始将不同指标之间的比较和相关性纳入其中，作为一种寻找其隐藏关系的方法是有意义的。

虽然我不确定它是否会在近期内实现，但我希望看到增强的最后一个方面是自定义指标外观的能力。当前的定制和样式功能几乎不存在。除了在指定 *value going up* 时包含颜色指示器外，没有有意义的方式来用颜色、标签、形状或详细程度来定制图表。特别是，在移动版本上显示的图表可能会缺乏具体性，无法看到值的范围和大小，反而更倾向于展示类似 sparkline 的外观，这种外观旨在快速传达数据的形状或趋势，但并不适用于识别确切值。如果未来涉及将 Pulse 指标集成到工作流程中，那么它们应该从美学上无缝集成。

## Tableau Agent

Tableau 和 Salesforce 无论如何都不会很快回避 AI 辅助的概念。事实上，可以说他们正在加倍投入这个概念，并展望一个 AI 不仅作为助手，而且可以自主行动的未来。这与 Tableau 最新的品牌信息“AI 驱动的分析平台”相一致。因此，在短期内，Tableau Agent 提供的 AI 辅助预计将迅速扩展。

尽管仍在开发中，Tableau 已经提到了 Tableau Agent 用于构建仪表板。我想象这将是基于一个更具分析导向的提示（例如，“为我制作一个仪表板，以便跟踪和分享季度销售业绩”）同时创建多个图表。也许这个功能将消除使用单个工作表的需要，并开始从仪表板画布上开始创作体验。

Tableau 还提到，Tableau Agent 的未来版本将能够承担更多的咨询角色。这意味着从纯粹的任务导向辅助扩展到包括可视化最佳实践、图表类型建议以及分析某些类型数据（如客户数据）的常见方法的潜在包含。我还听说提到了 *检索增强生成*（RAG）。RAG 是对 LLM 的增强，允许模型构建者将可信的外部来源纳入更详细的信息，尤其是当主题极其技术性时。因此，可能有一个未来，其中互联网上主题专家创建的内容的丰富性将被包含并通过 Tableau Agent 提供服务。

我希望以任务为导向的辅助在能力上能扩展得更远。目前，Tableau Agent 在各种工作表架子上构建计算并移动字段，但并没有减少格式化或设计的繁琐。包括这种类型的功能可以真正加快创建速度并最小化确保图表和仪表板内一致性这一痛苦的任务。最简单的例子就是能够指定所有元素的字体和字体大小层次；这可以节省大量的时间和精力。

其他行政任务也准备使用 Tableau Agent。在内容管理方面的所有事情都能从 AI 执行大量操作中受益，例如生成描述、标记内容、删除过时内容、突出重复内容、指定用户权限等等。在一个治理似乎成为更加突出主题的未来，让管理员能够以无代码方式大规模执行这些类型的操作将非常有价值。

最后，我认为 Tableau Agent 将进一步扩展到数据准备领域。通常被认为是最具挑战性的分析部分，数据准备包括寻找、清理、合并和塑造数据集以供使用的任务。如果 Tableau Agent 能够处理基于语言的请求，而这些请求传统上需要高级查询或编码能力，那么组织和分析师将能够从他们捕获的更大比例的数据中提取更多价值。

# 人工智能与数据分析

如第一章所述，其他分析和 BI 平台似乎也在走与 Tableau 相似的路径：高度依赖 AI 来提高生产力，降低构建分析内容的技术门槛，并自动化数据驱动的洞察的呈现以采取行动。这意味着经验丰富的分析专业人员将开始培训、调整和监督由 AI 创建或协助的内容。

数据专业人员如数据工程师、分析师和数据科学家的传统角色可能会分别转变为数据聚合者、提示创建者和模型训练师。例如，数据工程师可能不再需要依赖他们广泛的访问、摄取和组织数据源的技术知识，而是需要专注于确保 AI 能够访问组织所有数据的任务。这可能涉及更密集的治理工作，以及摄取*暗数据*——即在组织内部没有以有意义的方式被利用的数据，以扩展 AI 的知识。

未来的分析师可能不会花太多时间回答问题或为商业用户构建数据产品，而是可以基于业务领域构建语义模型，并制定创建分析提示的指南。最终，那些花费时间构建分析模型以解决问题的数据科学家可能会失去一些之前作为其角色关键组成部分的创造力和创新。他们可能不再需要自己构思新的模型，而是可能需要仔细审查、维护和改进由 AI（或 AI 模型本身）和训练数据提供的模型，随着其扩展到商业的更多方面。

虽然数据获取更多和更有意义的分析的未来听起来很理想，但通往这一目标的道路上有许多痛点，这些痛点与过去 20 多年中分析领域存在的痛点相似。首先和最重要的是合理化和收集所有组织数据。这个移动的目标已经经历了无数次的迭代和解决方案，从像 Hadoop 这样的存储解决方案到像 Snowflake 这样的云数据仓库，给人一种数据项目永远处于向最新和最伟大平台迁移的状态的感觉。AI 将为这一过程增加一个新层次，希望最终能够看到数据治理、管理和组织的成熟。

在人工智能能够为数据处理带来的支持和协助下，我满怀希望，这只是时间问题，关注点将重新转向利用组织数据的内涵，而不是永无止境的数据系统迁移游戏。同样，人工智能有望真正扩大分析思维到日常知识工作者，并使基于数据的行动真正增加。从某种意义上说，我是在谈论回归分析的基本：更好地理解和明智的决策。

# 人工智能的未来

在短期内，人工智能将继续扩大其影响力，尤其是在商业世界。我们现在正处于一个过渡阶段，为了提高人工智能的有效性，更多的知识将不得不投入到模型中。一旦这一消费阶段成熟，我们可能会进入一个精炼和增加人工智能自主性的阶段。人工智能自主性意味着低复杂性的任务、交互和请求将无需任何人类参与。

此外，随着人工智能的持续进步，专业化可能会增加。就像我们今天所拥有的“一切皆应用”的文化一样，可能存在针对狭窄领域的人工智能模型。时间将证明这些新模型是从头开始构建的，还是在内部构建的（就像 2000 年初的定制软件解决方案趋势一样），或者像 OpenAI 这样的 AI 组织是否会提供现成的解决方案，这些解决方案可以调整以满足商业需求（同时仍保持数据隐私）。

在非商业领域，人工智能的增加也应导致个人创造力和表达能力的提升。内容创作，如生成艺术和视频，已经变得极其流行，这一趋势有望随着社交媒体组织如 Meta 和 X 将人工智能助手整合到其平台中而扩大。

最后，随着苹果在 2024 年最后一个季度发布其第一代集成人工智能工具的 iPhone iOS 18 操作系统，人工智能助手以网络搜索、个人行政任务和信息聚合的形式无疑将成为主流。
