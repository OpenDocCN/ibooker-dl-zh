# 前言

我无法掩饰我的兴奋。2022-2024 年期间是现代技术时代最令人惊叹的时刻之一。有人称之为人工智能的“iPhone 时刻”，很多人现在正在发现人工智能的实际潜力。但我不认为这就是全部。我相信我们正进入一个指数级增长阶段，所有技术进步都发展得如此之快，以至于难以跟上它们的步伐。但这却是美妙的事情。几个月的进步和行业竞争，现在只需要几个月。曾经被认为不可能（甚至神奇）的事情，现在已经成为现实……而且这只是刚刚开始。

正是那种创新和彻底颠覆的感觉，让我第一次尝试 Azure OpenAI Studio 时感到如此。作为微软的一名 AI 专家，我获得了早期访问权限。这是一个非常早期的版本，绝对不是我们现在拥有的 Studio 和相关功能以及模型，但它非常有潜力。我们当时并不知道这个云服务即将成为生成式 AI 时代的超级巨星。而且这是一个现实，不是一个概念或未来的产品。这是我们用来创建我们自己的 GPT 风格实现的方法，具有不同的模型和成本/性能权衡，同时具有相对较低的实施和部署复杂性。

经过几个月的测试和跟踪新功能，OpenAI 发布了 ChatGPT。砰！我从未见过如此病毒式的与 AI 技术相关的时刻。即使在微软，每天晚上和同事们无数次的“极客”讨论中，都能感受到见证非凡事物的感觉。关键的时刻是 Azure OpenAI 服务中“聊天”功能的宣布，这使得任何公司都能首次测试和部署一个类似 ChatGPT 的实例，用于自己的目的。然后是 Bing Chat（它将演变成我们今天所说的 Microsoft Copilot）。砰！×2。这是我们第一次看到经典搜索引擎与 GPT 聊天体验的结合，在同一屏幕上……而且它真的工作了！人们可以直接得到他们精确需求的答案，而不是通过关键词搜索信息，然后自己确定正确的答案，他们用简单的语言做到了这一点。不是关键词，不是复杂词汇的组合。只是询问信息并等待答案。

几个月后，我们开始使用 Azure OpenAI 部署第一个概念验证。我是现场团队的一员，所以我非常接近采用者的现实——他们对生成式 AI 的理解、他们设想的使用案例、他们的担忧等。我也是微软 AI 社区的一员，这个社区充满了能量和创造力，去探索新的方法、发现新的架构、了解最新的技术和加速器。相信我，那一刻，不止我一个人感到幸运。这是纯粹的能量。

在某个时候，我猜想这可能是因为我作为大学教授的学术背景，我感觉大量的信息——虽然对任何学习者或采用者都非常有用——但对于任何试图开始使用生成人工智能和 Azure OpenAI 服务的公司或个人来说，也可能有些令人不知所措。全世界有大量的需求，这项技术应该以安全、负责任的方式被大规模采用。

那时我开始草拟使用 Azure OpenAI 服务进行应用开发的技术指南的主要概念。最初，这只是记录我不断收集的所有 URL 和信息的一种方式。然后，我继续根据我自己的实施经验添加笔记。最后，我根据从客户、朋友甚至家人那里不断收到的疑问和讨论不断更改或添加内容！

这是一个很好的基础，我知道它可以成为官方的技术指南，甚至是一本完整的书。我决定与我的 O'Reilly 同事交谈，并提出这个主题。这些对话只花了几个星期。潜力是显而易见的，但挑战巨大：在尽可能短的时间内创建高质量的 O'Reilly 级别的内容，以便所有生成人工智能的采用者都能开始阅读和学习。

这是我最具挑战性但仍然充满回报的经历之一。我感到非常荣幸能写这本书。世界上有那么多微软员工可以完成这项工作，因此我非常认真地抓住了这个机会。我的主要目标是创建一个包含所有关键元素，以适应（不断）变化的环境——展示最佳功能和实施方法，但同时也知道很快就会有其他方法，以及生成可用性和新功能的持续变化。但这正是魅力所在，也是我喜欢这本书及其背后的创作过程的原因。

我最喜欢的事情之一（我希望你也喜欢）是将书籍的典型静态内容与在线存储库的交互性、对不断发展的文档的引用以及嘉宾访谈的惊人力量结合起来。从一长串人工智能专业人士名单中汲取如此多的才华和知识，对于读者和热爱学习的你来说，是一种真正的奢侈，对我这个人工智能专业人士来说也是如此。

现在，我希望如果你已经决定开始阅读这本书，你已经准备好探索每一部分，从核心技术方面到其他相关的商业和伦理方面，这些都将帮助你进行第一个使用 Azure OpenAI 服务的生成人工智能项目。

真诚地，

Adrián

# 本书是如何组织的

本书的内容组织方式遵循了新技术典型采用工作流的顺序：对它们潜力的初步理解、技术实现的探索、实施考虑因素以及业务需求。根据公司和其成熟度水平的不同，事情发生的顺序可能会变化。例如，经验丰富的 AI 团队将更清楚地了解业务方面和未来的实施，然后巩固技术部分。无论你和你公司的背景如何，七本书章（加上一个附录）都应该涵盖你利用 Azure OpenAI 服务为你的生成式 AI 实施所需的内容：

第一章，“生成式 AI 和 Azure OpenAI 服务的介绍”

人工智能、生成式 AI 以及 Azure OpenAI 服务在企业级实施中的作用 101 概述。如果你是从零开始你的生成式 AI 之旅，这是理想的。

第二章，“为生成式 AI 设计云原生架构”

基于云原生原则构建生成式 AI 应用的自上而下方法，包括来自 Microsoft Azure 云的最相关构建块。在开始探索 Azure OpenAI 服务之前的关键预备步骤。

第三章，“使用 Azure OpenAI 服务实施云原生生成式 AI”

这是探索不同 Azure OpenAI 接口的核心章节，包括视觉游乐场和 API，以及主要的实施方法和模式。

第四章，“额外的云和 AI 能力”

完美补充了第三章。如果你想要了解所有相关的“部件”，如向量数据库、编排引擎和其他与 Azure 相关的服务，这里是你要去的地方。

第五章，“实施生成式 AI 实施”

在我看来，这是最重要的章节，如果你想要了解“真实生活”的生成式 AI 实施意味着什么。你可以设计一个精彩的架构，充分利用 Azure OpenAI 和其他服务，但重要的是要实施所有必要的措施来确保、扩展、保护和优化你的部署。如果你是为公司创建生成式 AI 应用，这是必须的。

第六章，“详细阐述生成式 AI 商业案例”

即使您掌握了与 Azure OpenAI 相关的所有技术方面，您仍然需要从业务角度使其工作。这意味着创建可持续的业务案例，并辅以现实成本估算和项目路线图。最终，如果这些话题没有提前讨论，任何 AI 系统都不会被采用。

第七章，“探索大局”

使用微软技术概述生成式 AI 系统的未来状态，同时采访了一些行业顶尖人才，为您提供来自一线人士的关键见解。

附录 A，“其他学习资源”

为您提供的一组资源，以扩展您的学习体验。

我的目的是让这一系列章节为您提供对当前生成式 AI 实施的全景视图，这将使您能够带着所有所需的知识开始新的项目。
