# 前言

> 风浪总是站在最有能力的航海者这边。
> 
> 爱德华·吉本

我已经编写代码很长时间了——比我想承认的时间还要长——我见证了多次横扫软件开发行业的风浪（并对其中的许多进行了写作或培训）。从互联网到 CI/CD 到容器到云，范式转变颠覆了旧的工作方式，催生了创新、令人难以置信的工具以及我们创建软件方法的转变。而且，从广义上讲，这些转变也产生了不确定性和困惑，因为个人试图理解它们并找出如何适应和应用他们的技能。

现在我们有了生成式 AI，这可能是我们在技术领域遇到的最大变革浪潮，尤其是对于软件开发者来说。就像其他一样，它正在激发巨大的创新和推动巨大的变化。但在某些方面，它更像是海啸，新的工具、新的模型和新的能力展现出巨大的力量，而编码者则被卷入其中，试图保持我们的集体头脑清醒。

这本书是我尝试帮助你冲浪——至少是在使用 GitHub Copilot 这个 AI 巨头的方面。我不会声称它将回答你所有的问题，但希望它能帮助你达到一个非常舒适的工具使用水平，让你了解它的能力（从基础到高级主题），并激发你使你的编码任务比以往任何时候都简单和容易。

我已经使用 Copilot 多年，以多种方式：在个人项目中，作为前 DevOps 总监，在一家企业研发组织中试点它，作为多个供应商的 Copilot 培训师，作为一个怀疑论者，以及作为一个热情的爱好者。关于最后两个角色，我在 Copilot 的早期并不像对其他工具那样印象深刻。但随着这本书的撰写、相关课程的开发以及观察它的演变，它已经成为我在集成开发环境（IDE）或 GitHub 中日常工作时使用的主要工具之一。Copilot 已经取得了长足的进步，甚至在过去的几个月里。在功能和功能方面取得了一些令人印象深刻的进展。我试图在书中捕捉到尽可能多的这些进展。

我在撰写这些章节的一个目标就是分享使用 Copilot 的好处和挑战，并尽量帮助你避免惊喜。所以，在你深入之前，这里有几点需要注意：

+   在过去一年左右的时间里，GitHub Copilot 几乎在每次发布中都至少改变了一个对话框或控制的位置。我已经尽力在写作时更新截图和描述。但可以肯定的是，有些东西在你阅读时可能已经被移动、外观改变等。

+   Copilot 现在正以相当规律的速度发布新功能。因此，随着时间的推移，书中将不会展示一些新功能。同样，截至写作时，Copilot 的一些功能被标记为*预览*或*实验性*。这些功能可能在你阅读本书时已经转移到生产环境中，或者如果它们没有达到标准，可能已经被移除。

+   Copilot 在许多不同的环境中运行，包括多个 IDE。在每个 IDE 中，都可能存在（并且确实存在）集成、外观甚至功能上的差异。GitHub 历史上一直优先考虑 Visual Studio Code 以及使用相同界面（如 GitHub Codespaces）的环境来推出新功能和包含功能全集。因此，本书中的所有截图和示例都是使用 VS Code 或 Codespace 完成的。

+   对于使用非 VS Code IDE 的用户，一些功能或功能可能尚未在 GitHub 的该环境中实现。请查阅 Copilot 和/或 IDE[文档](https://docs.github.com/en/copilot)以获取最新信息。

+   类似地，与 Copilot 集成的 IDE 通常可以在包括 Windows、Mac 和 Linux 在内的多个平台上运行。由于在书中涵盖所有变体不切实际，示例都是在 Mac 上完成的。因此，显示的窗口和控制可能反映了 Mac 风格。此外，不同平台之间的键盘快捷键可能会有所不同。为了避免每次引用键盘快捷键时都要列出所有快捷键，我们采用了*Meta*作为表示要在您的平台上插入适当快捷键的符号。

所有这些内容都适用于本书的整个结构，本书旨在从*它是什麼*的基础知识到高级*如何超越核心功能*进行讲解。大多数章节可以独立阅读，你应该可以根据自己的需求和舒适度自由跳读章节。

# 本书结构

由于你在阅读这本书，我想你对于 GitHub Copilot 作为一个潜在的 AI 编码助手或者至少想要了解更多关于如何使用它以及类似的编码助手来帮助你更快、更有效地工作感兴趣。

为了帮助理解，以下是本书结构的概述。

章节 1、2 和 3 指导你了解 Copilot 是什么以及如何在其两种主要模式——代码补全和聊天（会话）界面——中与之交互所需的基础知识。

第四章探讨了与 Copilot 进行更自主编辑的高级交互方式，以及使用代理模式从提示到完成自动驱动更改。本章还探讨了 Copilot Vision 的独特界面以及使用 Copilot 进行调试。

第五章和第六章展示了你如何利用 Copilot 创建丰富的测试集和多种类型的文档，以简化这些任务，并为你提供更多时间专注于其他编码任务。

第七章专注于如何提高 Copilot 结果的及时性和相关性。

第八章探讨了如何使用 Copilot 处理一些不太常见但有用的任务。

第九章探讨了 GitHub 中 Copilot 的聊天界面以及 Copilot 如何帮助你更好地了解你的项目，并简化与 GitHub 问题和拉取请求的协作。

最后，第十章提供了通过集成 GitHub 扩展来增强 Copilot 功能的信息，并展示了如何构建你自己的。

当你开始阅读时，你不需要了解任何关于 Copilot 的知识来使用这本书。但我确实假设你熟悉基本的编码技能，并且对 Git 和 GitHub 有基本的操作知识。下一节将更详细地解释可能从这本书中受益的读者类型。

# 目标读者

这本书是为那些试图了解更多关于如何使用 AI 助手来开发软件的人而写的。是的，它特别关于 GitHub Copilot，但许多示例和流程在其他 AI 编码工具和环境中也存在相似之处。为了最大限度地利用这本书，你应该已经具备在 IDE 中编码的经验，并且对 GitHub 有基本的了解。明确地说，这本书不会教你如何编码，如何使用 IDE，或如何使用 GitHub。但它将通过使用 Copilot 来帮助你在这所有领域提高你的生产力。无论你是软件开发者、质量工程师、SRE，还是那些想要了解 AI 助手如 Copilot 如何对他们有益的人，我相信你会找到有意义的信息和见解。

在撰写本书时，我考虑了一些受众群体，我认为他们可以从这本书中受益：

+   对于那些新接触（或刚刚接触）AI 编码助手并想要了解它们是什么以及如何最好地利用它们的人来说

+   那些已经理解了使用 AI 编码助手的理念和流程，并想要了解 GitHub Copilot 能提供什么的人

+   那些想要了解和评估 GitHub Copilot 作为在他们的组织或业务中更广泛使用的一种可能工具的人

+   那些已经有一些 GitHub Copilot 经验并想要充分利用其功能和功能的人

+   那些想要了解如何弥补 GitHub Copilot 的一些不足的人

+   那些想要了解 Copilot 的新功能，如代理模式的人

+   那些想要创建自己的 GitHub Copilot 扩展的人

+   那些与 GitHub 合作并想要利用 Copilot 集成的人

如果其中之一符合您的用例，我希望这本书能为您提供您所寻找的价值。如果您阅读了它，并且有机会，通过评论或在未来的会议或培训场所的互动，反馈总是受欢迎。关于如何在编码中驾驭生成式 AI 浪潮的信息从第一章开始。祝您冲浪愉快！
