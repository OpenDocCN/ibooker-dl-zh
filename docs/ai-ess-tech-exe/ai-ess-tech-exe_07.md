# 第七章：为人工智能雇佣正确的团队

高管们犯的另一个代价高昂的错误是错误地雇佣人工智能团队。这个错误可能导致资源浪费、团队流失和人工智能努力的延迟。

# 人工智能专家陷阱

许多高管急于立即雇佣数据科学家或机器学习工程师（MLE）。虽然这些角色至关重要，但过早地引入他们是一个典型的“脚枪”场景。以下是原因：

数据科学家和机器学习工程师（MLE）需要数据来工作。如果你还在产品发现阶段，他们通常没有什么事情可做。

承担这些角色的人可能没有能力从头开始构建全栈产品。缺乏明确的产品方向，他们可能会无意中使你偏离轨道。

# 更智能的雇佣步骤

在构建人工智能产品时，不要立即寻求人工智能专家，而是遵循以下一般步骤：

1. 专注于构建产品

首先，雇佣应用开发者，他们可以帮助快速迭代产品概念。在这个阶段，你仍在确定你的产品是什么，需要构建一个论点。

2. 建立数据基础

仪器化你的系统以收集正确数据并设置适当的可观察性。这可能需要平台或数据工程师，具体取决于你正在处理的数据类型和规模。

3. 优化你的 AI 系统

一旦你有一个工作产品和数据，那么你应该雇佣 MLE 或数据科学家来优化系统。在这个时候，他们应该能够设计指标、建立评估系统、运行实验和调试随机系统。

关键是保持在每个阶段都有强大的领域专家参与。

另一个不错的选择是引入顾问而不是雇佣全职专家。引入外部帮助可以帮助你完成上述三个步骤，同时避免全职雇佣的承诺。

# 正确雇佣方式

当你达到雇佣人工智能专家的阶段时，要有策略。

通过给候选人提供混乱的数据集来测试他们的数据处理能力。这项关键技能有助于设计有效的指标和评估。要深入了解，请阅读 Jason Liu 的博客文章[“10 Ways to Be Data Illiterate (And How to Avoid Them)”](https://oreil.ly/WRzNQ)。
