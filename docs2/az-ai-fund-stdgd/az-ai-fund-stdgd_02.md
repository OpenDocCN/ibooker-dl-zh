# 第二章\. Azure AI 服务

Azure AI 服务包含了许多工具，可以帮助你自动化语言处理、图像识别、智能搜索和内容生成等任务。令人兴奋的是，你不需要成为 AI 专家就可以深入其中。这些服务易于在 Web 和移动应用程序中实现，并且有十多种 AI 功能可供使用，你可以单独使用它们或混合使用。

Azure AI 服务最好的事情之一是它们是预构建并随时可用的。由微软高性能的 Azure 平台提供支持，这些预训练的 ML 模型对所有规模的企业都可用。所以，无论你是小型初创企业还是大型企业，你都可以利用微软多年的 AI 研究，而不需要成为技术高手。虽然许多工具可以直接使用，但其他工具——如 Azure AI 视觉、Azure AI 语音和 Azure OpenAI 服务——可以通过您自己的数据进行调整以满足特定需求。

另一个酷炫的功能是你可以轻松地将 Azure AI 服务集成到你的项目中。开发者可以通过表示状态传输（REST）API 或 SDK 访问这些工具。*应用程序编程接口（API）*允许不同的软件应用程序相互连接和通信——想象它们是你的应用程序之间的数字桥梁。例如，当你在线预订航班时，航空公司的网站使用 API 从其数据库中检查可用座位和价格。同样，当天气应用程序获取最新预报时，它会与天气服务的 API 通信以实时检索数据。*软件开发工具包（SDK）*另一方面，捆绑了开发者构建特定平台应用程序所需的所有工具、库和文档。这使得整个过程更加顺畅。此外，Azure AI 服务与 Azure Logic Apps 和 Power Automate 等其他 Microsoft 服务集成。这使得 Azure AI 服务更加灵活。

对于 AI-900 考试，你不需要深入研究这些服务的技术细节。但你应该对它们的功能有一个很好的理解，这正是本章我们将要探讨的内容。

# 微软 Azure 基础知识

微软 Azure 是一个拥有超过两百项服务的庞大云平台，可以帮助你在不同的环境中构建、运行和管理应用程序。以下是一些你可以选择的快速查看选项：

云

这可能就是当你想到 Azure 时脑海中浮现的画面。它是一个由远程服务器组成的网络，你的应用程序和数据就存储在这里，让你可以从任何地方访问你的资源。当需求激增时，你可以快速扩展；当事情放缓时，你可以缩小规模——所有这些都不需要担心硬件问题。

本地

如果你需要出于法规或安全原因在本地保留某些数据，Azure 提供了在您自己的本地服务器上运行其服务的灵活性。在这种配置中，你管理本地基础设施。

边缘

这涉及到在数据创建的地方附近处理数据，例如使用 *物联网 (IoT)* 设备或在偏远地区。Azure 的边缘服务让你可以降低延迟和带宽成本，同时保持一切运行顺畅，即使你的互联网连接并不总是可靠的。

Azure 几乎满足所有技术需求。无论是使用 Azure Blob Storage 和 Azure Cosmos DB 进行数据存储，还是使用 Azure Virtual Machines 和 Azure Kubernetes Service 运行 *虚拟机 (VMs)* 和容器，都有适合你的解决方案。而且，Azure 拥有超过 60 个区域和 300 个数据中心，其规模无与伦比。

请记住，微软偶尔会更新其服务名称或更改 Azure 用户界面。这意味着这本书中的一些步骤或视觉元素可能看起来略有不同。为了帮助解决这个问题，我建立了一个 [网站](https://oreil.ly/L22C3)，你可以在这里找到任何必要的更新。

# 设置

要设置 Azure 账户，只需访问 [*www.azure.com*](http://www.azure.com)。如果你是第一次使用 Azure，你可以注册一个免费账户。在第一个月，你将获得对整个服务目录的完全访问权限，并获得 200 美元的信用额度来探索 Azure 提供的一切。

此外，Azure 还为你提供某些服务的免费月度使用，尽管有一些限制。例如，Azure Machine Learning 和 Azure 认知服务等 AI 服务在前 12 个月内免费。有了这些服务，你可以构建、训练和部署机器学习模型，或者将语音、视觉和语言等 AI 功能添加到你的应用程序中。在试用期间，你可以在免费使用限制内尝试 Azure 认知服务文本分析或 Azure AI 视觉等服务。

即使在你第一年之后，超过 55 个服务只要你不超出使用上限，仍然免费。其中一些也专注于 AI，例如 Azure Functions 用于运行事件驱动代码，以及 Azure Kubernetes Service 用于管理容器化的 AI 工作负载。

如果你是一名学生，Azure 提供了一个学生账户，包含 12 个月的 100 美元免费信用额度——无需信用卡。

Azure 使用按使用付费的模式。一旦你的 30 天免费试用或 12 个月的 学生计划到期，如果你想继续使用服务，你需要切换到按使用付费，但任何剩余的信用额度仍然可以使用。Azure 还有一个 [定价计算器](https://oreil.ly/gF8eP)，并允许你设置预算和成本警报。没有长期承诺，你可以在任何时候取消。

对于本书中我们将要进行的操作，免费和学生版本将足够你开始。

如果你的雇主有 Azure 的订阅，你应该获得批准来为准备考试使用它。还重要的是要注意，你的雇主可能会限制对某些服务的访问。因此，你可能仍然需要设置自己的账户。

# 创建 Azure AI 资源

要使用 Azure AI 服务，您首先需要设置一个资源。有两种选项可供选择：

多服务资源

这允许您使用单个密钥和端点访问多个 Azure AI 服务。*端点*是您的应用程序连接到以与服务交互的 URL。这种设置非常适合探索各种服务，如 AI 语音、AI 视觉和 AI 语言，或者需要使用多个服务一起的项目。此外，它通过将所有服务合并到一个资源下简化了计费。

单服务资源

这是在您只需要一个特定的 AI 服务时使用。每个服务都有自己的密钥和端点。这使得跟踪使用情况和成本变得容易。

您应该选择哪一个？大部分情况下，这取决于您要构建的内容以及您如何管理成本。

有不同的方法来创建资源，但最简单的方法是通过 Azure 门户。只需登录，点击“创建资源”，在市场搜索“Azure AI 服务”以创建多服务资源。如果您只需要单服务资源，搜索特定的服务，如 AI 面部、AI 语言或 AI 内容安全。

一旦您选择了服务，点击创建。您需要填写以下详细信息：

订阅

这是您的 Azure 资源的计费账户，它决定了您将如何支付服务费用。

资源组

这是一个容器，帮助您在 Azure 中组织和管理相关资源。

地域

这是您的服务将被托管的地域。这将确保最佳性能并符合法规要求。

定价层

这是您选择的服务级别，它定义了与您的资源相关的功能、性能和成本。

您还需要为资源提供一个独特的名称以供识别。图 2-2 展示了此屏幕的外观。

# 工作室

在创建 Azure AI 资源后，您可以使用它来构建应用程序。您有几个选择——您可以与 REST API 或 SDK 一起工作，但还有一种更简单的方法：使用工作室。这些工作室使您无需具备深厚的编码技能即可轻松实验 Azure AI 服务。它们具有用户友好的界面，允许初学者和经验丰富的开发者探索、演示和评估服务。以下是一些您可以使用的工作室：

视觉工作室

这让您可以使用图像和视频分析工具。您可以尝试对象检测、图像分类和视频分析等功能。

语言工作室

此平台专注于基于文本的 AI 服务，如情感分析、翻译和文本摘要。它还提供了定制语言模型的工具。

语音工作室

在这里，您可以探索 Azure 的语音转文本、文本转语音和实时翻译服务。如果您正在寻找需要语音交互的应用程序，这是一个非常有用的工具。

内容安全工作室

该工作室旨在管理和检测有害内容，如暴力或仇恨言论，它可以帮助您设置防止不适当内容的防护措施。

除了这些单独的工作室之外，还有 *Azure AI Foundry*（以前称为 *Azure AI Studio*），它将多个 Azure AI 服务整合到一个平台上。开发者可以使用这个基于网络的平台，使用最新的 API 和模型来创建 AI 应用程序。

要了解其工作原理，让我们通过使用 Vision Studio 的快速演示来操作。您可以在 Azure Portal 的“计算机视觉”下找到它。如果您看不到，只需使用搜索栏来定位它。

一旦您点击“计算机视觉”，选择“创建计算机视觉”。您将被带到配置页面，您需要填写一些详细信息：

订阅

选择您的 Azure 计划，例如免费试用或按使用付费。

资源组

如果您已经有一个资源组，您可以在此处选择它。否则，点击“创建新”并输入组名（例如，“AI-Vision-Group”）。

区域

选择您的计算机视觉资源托管的地域。选择靠近您的区域可以帮助减少延迟并提高性能。

名称

为资源输入一个描述性的名称（例如，“AI-Vision-Service”）。这使您更容易管理和跟踪资源。

定价层

选择适合您需求的定价计划——无论是免费层还是具有更大配额的更高级选项。这与订阅有何不同？您的订阅定义了您支付所有 Azure 服务的方式，而定价层定义了此特定服务的具体成本结构。

一旦填写完所有内容，请检查您的设置并点击创建。资源设置完成后，选择“转到资源”，然后点击“转到 Vision Studio”。您可以在图 2-3 中看到 Vision Studio 仪表板的样子。

# 结论

Azure AI 服务非常强大。该平台提供了一套灵活的工具，允许您轻松地将高级 AI 功能集成到应用程序中。这些服务旨在简化复杂任务，使 AI 对所有人可访问，无论他们的技术背景如何。

但在准备 AI-900 考试时，重要的是要知道，你不需要对 Azure 技术有深入的技术理解就能通过考试。考试更侧重于高级概念，而不是工具如何工作的复杂细节。
