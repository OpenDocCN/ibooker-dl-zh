# 前言

我在 20 世纪 80 年代初开始编程。我的第一台设备是 Atari 400——它的膜式键盘并不算强大，只有 8K 的 RAM，程序是从磁带加载的。我学习了 BASIC 语言，并创建了简单的游戏和实用应用程序。

我对此完全着迷。

自然地，随着时间的推移，我升级到了更强大的机器。在这个过程中，我接触了像 Pascal、C 和 C++这样的语言。但 IDE 并没有太大变化，除了语法高亮和调试功能。

随着 GitHub Copilot 和 ChatGPT 的到来，一切发生了戏剧性的变化。尝试这些工具的感觉就像是我第一次拿到 iPhone 的那一刻——绝对具有颠覆性。

在自然语言中，我让 ChatGPT 编写代码。或者在 VS Code 中，我会输入函数的一部分，GitHub Copilot 就会生成代码块。很多时候，这就像打中了靶心。哦，然后我还用 ChatGPT 将图像转换为代码。

但真正的力量在于这些工具可以处理开发者许多繁琐的任务。谁愿意与正则表达式语句搏斗，或者拼接 bash 命令或 GitHub 动作？当然不是我了。但这些 AI 编程工具？它们对这种东西就像早餐一样。

结果证明，这些 AI 工具不仅对编程很有用。我开始使用 ChatGPT 来构思应用程序的想法，起草需求，甚至编写单元测试。

我很快就相信了，AI 辅助编程将是程序员必备的技能之一。

所以是的，我看到了写这本书的巨大需求。我整理了一个大纲，并向 O'Reilly 提出了建议。那里的同事们立刻看到了它的潜力。

写这本书很有趣，我也学到了很多。我还采访了许多聪明的开发者，他们为我提供了很多好主意和技巧。

但 AI 辅助编程正在迅速发展。这就是为什么这本书有一个配套的[GitHub 仓库](https://github.com/ttaulli/AI-Assisted-Programming-Book)。在这里，我会更新这本书，并包括这个激动人心的领域中的其他重要发展。

所以感谢您拿起这本书。我希望您不仅觉得它信息丰富，而且在您的旅程中也是一个有价值的指南。

# 涵盖内容

下面简要介绍一下每一章：

+   第一章，“开发者的新世界”：这一章从探讨生成式 AI 如何改变程序员的游戏开始。它讨论了这些 AI 工具如何帮助开发者更多地思考大局，而不是编码的细节。章节还回顾了编程语言的历史。还有关于 GPT-4 等高级 AI 技术的细节。

+   第二章，“AI 编码技术是如何工作的”：本章首先解释了生成式 AI 以及为什么变换器模型和大型语言模型在编程世界中如此重要。最后，通过 OpenAI 的 Playground 的演示，展示了如何与这些 AI 模型互动，并调整它们以满足您的编码需求。

+   第三章，“提示工程”：这里的信息对于使用 AI 辅助编程工具至关重要。本章充满了实用的技巧，例如处理冗长或令人困惑的提示，以及防止 AI 编造内容。此外，它还分解了提示的关键部分，并展示了如何有效地使用它们。

+   第四章，“GitHub Copilot”：本章是对这个强大工具的详细说明。其中包括了创建带注释的代码、Chat 功能和使用 AI 驱动的命令行界面等核心特性。还涵盖了为专有代码库定制系统的内容。

+   第五章，“其他 AI 辅助编程工具”：本章详细介绍了其他顶级 AI 辅助编程工具，如 Amazon CodeWhisperer、Google 的 Duet AI 和 Replit 等，仅举几个例子。

+   第六章，“ChatGPT 和其他通用 LLMs”：这部分内容涵盖了如何使用这些工具处理正则表达式、起始代码和 GitHub Actions 等任务。

+   第七章，“想法、规划和需求”：这里的重点是使用聊天机器人启动软件项目。这包括头脑风暴、市场研究、需求文档和测试驱动开发等主题。

+   第八章，“编码”：本章探讨了开发中的常见场景，无论是与 API 一起工作、使用模块化编程还是重构。还涉及了处理函数和面向对象编程的内容。

+   第九章，“调试、测试和部署”：本章讨论了开发中不那么光鲜的部分。它涵盖了修复错误、使用 AI 辅助编程工具进行代码审查、编写单元测试和描述拉取请求等主题。

+   第十章，“总结”：这是本书的总结，强调了主要观点。

# 这本书的不同之处

软件开发者依赖于确定性。当你向程序提供特定的输入时，你总是得到相同的输出。多年来，这种纯粹的确定性逻辑是软件的核心和灵魂。

但当您使用 AI 辅助编程工具时，事情会变得有些混乱。得到结果就像掷骰子一样，因为一切都是基于概率的。当您提示 AI 工具生成一些代码，即使多次使用相同的提示，每次也可能得到不同的结果。当然，一开始可能会让人感到困惑，但一旦掌握了技巧，就完全值得了。这就是为什么有一章专门介绍提示工程，以帮助您掌握这种新的编程方法。
