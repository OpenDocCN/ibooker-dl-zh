在本章中，我们将从一些策略开始，帮助你自信地应对考试。之后，我们将按类别逐一讲解关键主题，为你提供有针对性的总结，以指导你的学习课程。

# 考试时的技巧

考试表现的关键在于策略。当然，了解材料是至关重要的——但你如何应对考试可以产生很大的影响。管理你的时间、保持专注和使用聪明的应试技巧可以帮助你充分利用每个问题。在本节中，我们将介绍实用的技巧，帮助你在整个考试中保持敏锐和自信，从而将你的准备转化为及格分数。

## 管理你的时间

良好的节奏在考试当天可以产生很大的影响。你将有 90 分钟的时间来完成 65 个问题。这大约是每个问题一分钟，上下浮动。

从你立刻就知道的选项开始排除。这些快速胜利将建立你的信心并保持你的势头。如果你遇到一个难题，不要陷入困境。标记它以便复习，然后继续前进。专注于先回答你确定的问题。然后，在你的第二次遍历中，你将有更多的时间（和更少的压力）来应对更难的问题。

## 仔细阅读问题

这听起来很简单，但放慢速度仔细阅读每个问题可能是考试中的一种有效策略。许多问题在像 *not*、*except* 或 *only* 这样的小词中隐藏着重要的线索——错过它们可能会完全改变问题的含义。

例如，一个问题可能会问，“以下哪项不是使用 AWS AI 服务的优点？”那个小小的词 *not* 改变了含义——你正在寻找例外，而不仅仅是列出优点。错过它，你可能会轻易地选择错误的答案，甚至都没有意识到。

多花几秒钟仔细阅读可以让你更清楚地了解实际上在问什么。这种清晰度有助于你第一次就答对更多的问题，这样你将花费更少的时间猜测和更改答案。

## 使用排除法

当你遇到一个多项选择题并且不确定答案时，首先排除那些明显错误或不合逻辑的选项。去掉明显的异常值可以缩小范围。

这个技巧在正确答案不明显时特别有用。缩小你的选择范围能让你更有可能做出明智的猜测。比如说，你面对四个答案——如果你能排除其中一个，你猜对的几率就从 25%上升到 33%。排除两个，你就有了 50/50 的机会。

除了提高你的机会之外，排除法的过程还能节省精神能量。选项越少，猜测就越少，这有助于你保持敏锐并自信地通过考试。

## 保持冷静并仔细检查你的答案

在整个考试过程中保持冷静至关重要。当你遇到难题或注意到时间在流逝时，很容易感到压力。这时，你自然会感到压力增大，但让这种压力占据上风可能会导致匆忙的决定和错误。相反，花点时间深呼吸并重新集中注意力。几口深呼吸可以减缓你飞快的思绪，这有助于你恢复控制，并将冷静带回你的心态。这个暂停可能看起来微不足道，但它可以在你清晰地思考和高效地处理每个问题时产生巨大的影响。

如果你管理好时间并在考试结束时还剩下几分钟，利用这些最后的时间来回顾你的答案。回到任何你标记为需要复查的问题，尤其是那些你觉得棘手或最初匆忙处理的问题。这最后几分钟可能非常有价值。它们可以帮助你发现任何小错误或犹豫不决的时刻。通常，重新审视一个问题可以带来清晰，并帮助你做出更自信的选择。即使是微小的修正也能提高你的分数，所以利用任何额外的时间来仔细检查你的工作是明智之举。

如果你时间紧迫，需要在考试前的最后一小时最大限度地提高学习效率，这门速成课程就是为你准备的。以下章节将分解考试中可能出现的最重要的概念、工具和服务。把这看作是一次高效率的复习：不是完整的章节回顾，而是精华的提炼。

以下每个章节都专注于考试的某个不同领域，从机器学习和生成式 AI 的基础概念到支持负责任 AI、安全和治理的 AWS 服务。如果你已经完成了这些章节，你会认识到熟悉的主题，甚至可以用这作为地图来回顾关键部分。

## 人工智能和机器学习的基础

从高层次来看，机器学习（ML）是指教会系统从数据中学习并做出预测，而不需要明确编程。你需要了解主要机器学习类型之间的区别：

+   监督学习在标记数据上训练模型，其中结果已知（比如：预测房价）。

+   无监督学习在未标记的数据中寻找模式，比如根据购买行为对客户进行分组。

+   强化学习涉及一个代理通过试错来学习，根据其行为（如自动驾驶汽车学习避免障碍）获得奖励或惩罚。

理解过拟合和欠拟合至关重要。过拟合发生在模型在训练数据上表现良好但在新数据上表现不佳时——它是记忆而不是学习。欠拟合意味着模型过于简单，无法捕捉到潜在的模式。

你还需要了解特征工程——将原始数据转换为提高模型性能的输入的过程。把它想象成准备你的数据，以便模型可以更有效地学习。

当涉及到选择合适的机器学习技术时，上下文很重要。回归用于预测连续值（如房价），而分类将数据分配到类别中（如检测电子邮件是否为垃圾邮件）。聚类是一种无监督技术，它将相似的数据点分组在一起，而不需要标签。

预期会问到支持人工智能/机器学习的关键 AWS 服务。Amazon SageMaker 用于构建、训练、调整和部署机器学习模型。对于文本分析，Amazon Comprehend 可以识别情感、关键词和主题。在处理音频时，Amazon Transcribe 将语音转换为文本以进行进一步分析。

另一个必须了解的概念是推理。在训练好模型后，推理是指使用它对新数据进行预测的行为。它是模型在训练期间学习到的一切在现实世界中的应用。

评估指标也经常出现。对于分类任务，准确度衡量模型预测正确的频率。对于回归任务，如均方根误差（RMSE）之类的指标用于评估预测误差。

在模型部署方面，你应该了解批量推理和实时推理之间的区别。批量推理处理大量数据，而实时推理在数据到来时立即处理单个输入——这对于聊天机器人或欺诈检测等应用至关重要。

最后，确保你熟悉模型监控和诸如数据漂移和模型漂移等概念。随着时间的推移，输入数据的变化可能导致模型性能下降。像 Amazon SageMaker Model Monitor 这样的服务可以自动跟踪并标记这些问题。

## 生成式人工智能的基础

生成式人工智能指的是基于从训练数据中学习到的模式创建新内容（如文本、图像甚至代码）的模型。

你肯定需要了解 Amazon Bedrock，这是一个托管 AWS 服务，允许你构建生成式人工智能应用，而无需配置基础设施。Bedrock 为开发者提供了轻松访问顶级提供商的 FM（功能模块）的途径——无需深入的人工智能/机器学习专业知识。

接下来，你需要了解 FMs。这些是巨大的预训练模型，可以针对特定任务进行微调。微调——即使用特定领域的数据重新训练一个通用模型——对于将 AI 适应特定行业，如金融或法律服务，至关重要。

当处理文本数据时，你将看到术语*token*出现。Token 是模型处理的最小文本单元。这可能是一个单词、子词，甚至是字符。

另一个重要概念是非确定性。生成模型如 LLMs 并不总是输出相同的结果，即使你两次提出相同的问题。这使得模型具有创造性，但也需要仔细评估。

谈到评估，你将需要熟悉如下的指标：

+   用于检查文本摘要质量的 ROUGE 分数

+   用于评估翻译准确性的 BLEU 分数

接下来，你需要了解提示工程。这是一种构建清晰、具体的输入以引导模型输出的实践。例如，如果你想得到一个突出特点而不夸张的产品描述，你编写提示的方式非常重要。

一个常见的考试话题是幻觉。在生成 AI 中，幻觉意味着模型编造听起来令人信服但并不真实的信息。

此外，确保你了解 transformers，这是大多数现代 LLMs 背后的架构。Transformers 擅长处理文本序列和理解长段落中的上下文，这就是为什么它们在聊天机器人和摘要中如此有效。

最后，如果你被问到关于图像生成的问题，了解像扩散模型（逐渐将噪声图像细化成清晰的图像）和 GANs（在游戏设置中使用两个模型生成逼真图像）这样的技术之间的区别。

## FMs 的应用

Bedrock Agents 自动化复杂的 AI 工作流程，例如聊天机器人检索客户索赔或搜索内部文档的工具。这是通过 RAG 实现的。这种技术使用向量搜索。如果你正在使用嵌入（数据的数值表示），像带有 pgvector 扩展的 Aurora PostgreSQL 这样的服务让你能够高效地搜索相似项。

你将需要了解上下文窗口。当将提示输入到 FMs 中时，上下文窗口定义了模型一次可以处理多少信息。更大的窗口意味着模型可以“记住”更多，这对于像长篇对话或文档摘要这样的任务很重要。

当构建聊天机器人时，平均响应时间变得重要。如果你正在资源受限的设备上工作——如无人机或物联网设备——优化的小型语言模型（SLMs）是边缘上低延迟推理的解决方案。

另一个重要的话题是持续预训练，这意味着随着时间的推移，通过更新模型以新鲜数据来保持其输出的相关性。在金融或医疗保健等快速变化的领域，这有助于防止 AI 系统过时。

最后，要意识到像提示中毒和提示劫持这样的风险。这些问题是安全问题，恶意输入会诱使模型表现不佳或泄露敏感信息。

## 负责任的人工智能指南

负责任的人工智能的核心是公平性。如果一个模型在存在偏见的训练数据上训练——比如，一个过度代表某些群体的欺诈检测模型——可能会导致严重后果，如不公平对待或违规监管。一个平衡的数据集，公平地代表所有群体，对于构建无偏见的系统至关重要。

您还需要了解可解释性。AI 准确是不够的；决策者需要了解 AI 为何做出特定的预测。使用可解释模型或像 Amazon SageMaker Clarify 这样的工具，以简单术语解释决策有助于实现这一点。

当与 FM（功能模型）一起工作时，您需要管理像偏见、幻觉、剽窃（特别是如果 AI 生成的内容没有正确引用）和毒性（有害或不适当的内容）这样的风险。AWS 提供了像 Amazon Bedrock 的护栏这样的工具，以帮助实时过滤和管理 AI 输出。

另一个重要主题是内容审核。如果您正在构建聊天机器人或任何交互式应用，您需要防止有害或不适当的响应。审核可以通过内容审核 API 或内置的护栏来处理，这些护栏会在输出到达用户之前标记出问题输出。

当然，人类判断无疑是关键的。这就是人类在循环过程中的作用所在。这允许人们验证 AI 输出——对于图像标记等任务特别有用。

# 人工智能解决方案的安全、合规性和治理

AWS 拥有许多安全服务。IAM 帮助控制谁可以访问什么。AWS CloudTrail 记录每个 API 调用和访问尝试，使得检测未经授权的活动更加容易。如果您担心内部或外部威胁，Amazon Macie 会自动发现并保护敏感数据，如个人信息。

当涉及到合规性时，AWS 提供了 AWS Artifact 这样的工具，它为您提供了访问合规性报告和认证的权限。如果您需要持续监控您的云资源以符合合规性，AWS Config 会密切关注一切，以确保您的设置与内部政策保持一致。

另一个重要概念是加密和密钥管理。为了安全地处理模型工件（如训练模型或日志），您将使用 AWS 密钥管理服务（KMS）来管理加密密钥。始终确保 AI 模型、训练数据和输出日志被加密，尤其是如果您正在处理敏感或受监管的信息。

对于无需互联网暴露的可靠部署——这对于金融等行业至关重要——您将需要使用 AWS PrivateLink。它允许服务之间进行安全通信，而不会将流量暴露给互联网。

治理将一切联系在一起。治理意味着确保 AI 解决方案不仅安全且合规，而且可审计和透明。这就是为什么你应该使用像 Amazon SageMaker Clarify、AWS Audit Manager（审计证据收集的自动化）和 Amazon SageMaker Model Cards 这样的工具。
