# 第三部分 超越最小可行产品

可行产品

当利益相关者手中有了最小可行产品（MVP）时，您将开始收到反馈并考虑进一步开发的可能性，将 MVP 的功能扩展到令人兴奋的新方向。

第八章通过扩展系统的异常处理来增强 ACH 处理。软件的这个重要方面可能并不引人注目，但对于稳健且用户友好的软件来说仍然至关重要。

第九章通过提供搜索和创建审计跟踪的能力来扩展 ACH 仪表板，以跟踪用户与 API 的交互。

第十章将公司信息添加到仪表板中。这项活动将让您探索几个额外的领域，为增强仪表板开辟新的方向。

第十一章深入探讨国际 ACH 交易及其对我们系统产生的监管影响。

第十二章通过提供如何进一步推进这个项目的想法来总结一切。

阅读书籍后，您将熟悉各种技术、工具和技术。我们希望您已经找到了一些有趣的主题进行深入研究。此外，您将拥有一个相当稳健的 ACH 仪表板，并对 ACH 处理有更深入的理解！
