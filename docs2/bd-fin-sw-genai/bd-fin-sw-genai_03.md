# 第二部分：通往最小可行产品的道路

可行的产品

现在你已经了解了 ACH 文件的结构，是时候开始构建一个功能性的 ACH 仪表板了。如果听起来很令人畏惧，别担心！我们首先构建各种组件，然后将它们全部组合成一个最小可行产品（MVP）。

第三章从使用 Docker 设置环境开始。在这里，我们探讨了 Docker 的基本知识以及如何在构建项目时使用它。

第四章将使你通过利用 FastAPI 框架来熟悉在 Python 中使用 API。构建基本的 RESTful API 以支持仪表板，并使用 OpenAPI 探索文档将使你忙碌。

第五章讨论了在 Postgres 关系型数据库中存储 ACH 文件的数据库设计。你将有机会设计数据库，并看到不同设计的一些优点和缺点。

第六章探讨了使用 Next.js 创建用户界面，从后端过渡到前端。

第七章将前几章创建的所有组件汇集在一起，组成一个最小可行产品（MVP），你将能够将其交付给利益相关者。

在本部分的结尾，你应该拥有一个你可以引以为傲的 ACH 仪表板。这实际上是项目的起点。项目上线后，你将开始收到来自利益相关者的各种增强请求。
