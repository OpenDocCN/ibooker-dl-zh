# 为您的技能提升之旅提供相关 O’Reilly 书籍

这一系列书籍包括有助于您在公司层面上的阅读，同时也支持您的 Microsoft Azure 技能提升，特别是 AI 和生成式 AI 的技术细节。所有这些都与您在这里学到的内容相辅相成，您还可以关注作者的个人资料，继续从这些知名专业人士那里学习。让我们来探索它们：

*《AI 组织》*，由[David Carmona](https://oreil.ly/FOSlv)著

个人来说，这是我最喜欢的之一。这本书包括了 AI 采用公司的真实经验，以及从 AI 战略角度进行的专家访谈和经验教训。此外，David Carmona 是高级 AI 主题的绝对参考（以及微软同事）。

*描述*：这本实用指南解释了商业和技术领导者如何拥抱这种新型组织。基于真实的客户经验，微软的 David Carmona 涵盖了成为 AI 组织所需的旅程——从今天在您的业务中应用 AI 到能够使您的组织重新定义行业的深度转型。

*为什么选择这本书？* AI 采用的教训包括文化、技术、责任和商业话题，这些对于任何利用 AI 和生成式 AI 技术的公司都非常相关。坦白说，这本书是一部现代经典，也是各种职业人士，包括技术和商业人士易于阅读的资源。

[*《AI Ladder》*](https://oreil.ly/7bmtW)，由[Rob Thomas](https://oreil.ly/Ipe6t)和[Paul Zikopoulos](https://oreil.ly/1gCVc)著

另一部 O’Reilly 关于 AI 采用主题的经典之作，包含了采用公司的其他真实案例。

*描述*：IBM 的作者 Rob Thomas 和 Paul Zikopoulos 向 C 级高管和商业专业人士介绍了 AI Ladder，这是一种统一、指导性的方法，帮助他们理解和加速 AI 之旅。本书包含真实世界的例子和实际经验，探讨了 AI 的驱动因素、价值和机会，以及组织在采用过程中面临的挑战。

*为什么选择这本书？* 与《AI 组织》类似，这本书包括了 AI 采用的现实场景。但我最喜欢的是逐步的数据准备，这对于任何想要训练自己的 AI 模型或实施 RAG 或微调模式的人来说都是高度必要的。

[*《设计机器学习系统》*](https://oreil.ly/7UWAL)，由[Chip Huyen](https://oreil.ly/AFnId)著

在 AI 主题方面，他是最相关的作者之一。技术细节和易于理解的解释达到了很好的平衡。

*描述*: 在这本书中，您将学习设计可靠、可扩展、可维护且能够适应不断变化的环境和业务需求的机器学习系统的整体方法。

*为什么选择这本书？* Chip 创建了一个极好的资源，对于任何想要了解机器学习系统端到端操作的人来说，这是一本绝对参考书——评论也一致。适用于生成式人工智能和 LLMOps，如果您想获得一些技术知识和整体视野，我强烈推荐这本书。

[*高风险应用机器学习*](https://oreil.ly/acZdT)，由 [Patrick Hall](https://oreil.ly/b-WAX)、[James Curtis](https://oreil.ly/Jexy2) 和 [Parul Pandey](https://oreil.ly/CXpVc) 著

从高度应用的角度对负责任的人工智能主题的精彩介绍。

*描述*: 本书描述了负责任的人工智能方法，这是一个改善人工智能/机器学习技术、业务流程和文化能力的整体框架，它建立在风险管理、网络安全、数据隐私和应用社会科学的最佳实践之上。

*为什么选择这本书？* 我们在第五章和与 Sarah Bird 的访谈中涵盖了负责任的人工智能主题。但 RAI 是一个非常重要的主题，对于那些想要探索生成人工智能技术的潜力和考虑的人来说，它值得进行深入的探讨。

[*机器学习设计模式*](https://oreil.ly/xdbw_)，由 [Valliappa Lakshmanan](https://oreil.ly/UC8s5)、[Sara Robinson](https://oreil.ly/FoRMy) 和 [Michael Munn](https://oreil.ly/0I6en) 著

机器学习模型、相关 MLOps、可扩展性考虑等方面的介绍。

*描述*: 在这本书中，您将找到对 30 种数据与问题表示、操作化、可重复性、可再现性、灵活性、可解释性和公平性的模式的详细解释。每种模式都包括对问题的描述、各种潜在解决方案以及针对您的情况选择最佳技术的建议。

*为什么选择这本书？* 这是一本详尽的、生成式人工智能之前的指南，高度关注模型训练模式。它是《设计机器学习系统》的良好补充或替代品。强烈推荐作为具有技术背景的专业人士的额外学习资源。

[解码数据架构](https://oreil.ly/sqpJr)，由 [James Serra](https://oreil.ly/3CDHj) 著

了解从经典到湖屋架构的架构模式，这些模式可以为您的生成人工智能实现提供关键资源。

*描述:* 数据编织、数据湖屋和数据网格最近作为现代数据仓库的可行替代方案出现。这些新的架构具有显著的优势，但它们也围绕着很多夸张和混淆。这本书提供了对每个架构的导游，以帮助数据专业人士了解它们的优缺点。

*为什么选择这本书？* 詹姆斯·塞拉（James Serra），微软最有知识架构师之一，就足以成为任何人都应该阅读和推荐这本书的理由。这是一本更新后的指南，展示了现代公司数据平台所需的一切，包括为您的生成式 AI 发展提供动力的湖屋模式。

[*大规模数据管理*](https://oreil.ly/dV1Hk)，由 [Piethein Strengholt](https://oreil.ly/EJRyo) 编著

这本书是之前书籍的完美补充，特别关注联邦数据系统。

*描述*：随着数据管理不断快速发展，将所有数据集中在一个地方，如数据仓库，已不再可扩展。今天的世界是关于快速将数据转化为价值。这需要我们在联邦责任、数据管理和使其对他人可用的方式上实现范式转变。通过这本实用的书籍，您将学习如何设计一个下一代数据架构，该架构考虑了您组织所需的规模。

*为什么选择这本书？* 与詹姆斯·塞拉（James Serra）一样，皮特海因（Piethein）是这个领域的顶级声音，也是数据管理主题的巨大知识来源。这是与 CluedIn 的蒂姆·沃德（Tim Ward）讨论的延续，对于任何企业级实施都是高度必要的。另一本必读之作。

[*学习 Microsoft Azure*](https://oreil.ly/G2U08)，由 [Jonah Carrio Andersson](https://oreil.ly/gZGyw) 编著

来自社区 MVP 的 Microsoft Azure 云的精彩介绍。如果您想扩展对 Azure 的知识，超越这本书，则强烈推荐。

*描述*：如果您所在的组织计划将服务现代化并从传统的软件或本地私有云迁移到云端，这本书就是为您准备的。软件开发人员、解决方案架构师、云工程师以及任何对云技术感兴趣的人都将学习使用 Microsoft Azure 进行云计算、迁移、转型和开发的基本概念。

*为什么选择这本书？* 对于我来说，这是学习“所有 Azure 事物”的完美方式，如果您缺少核心基础设施、网络、管理、安全和其他主题的基础知识。乔纳（Jonah）也是这本书的技术审稿人之一。

[*Azure 食谱*](https://oreil.ly/lOREI)，由 [Reza Salehi](https://oreil.ly/W3lgM) 编著

一本包含逐步解释的技术指南，是之前 Azure 书籍的完美补充。这是扩展您对 Azure 知识，超越 Azure OpenAI 功能的好方法。

*描述*：您如何处理使用 Azure 时遇到的问题？这本实用的指南提供了超过 75 个食谱，帮助您在日常场景中处理常见的 Azure 问题。这包括设置存储账户权限、使用 Cosmos DB API、管理 Azure 基于角色的访问控制、使用 Azure 策略管理您的 Azure 订阅等关键任务。

*为什么选择这本书？* 对我来说，这是任何 Azure 学习者在阅读了 Jonah 的书之后的第二步。Reza 创建了一个广泛的食谱，对任何想要利用 Azure 资源的专业人士都非常有用。部署和管理步骤适用于 Azure OpenAI 和其他 AI 资源。

[*Azure AI 服务在云、移动和边缘的规模化应用*](https://oreil.ly/UbFc9)，由 [Simon Bisson](https://oreil.ly/xQwY2)、[Mary Branscombe](https://oreil.ly/JjlGg)、[Chris Hoder](https://oreil.ly/cR_Xk) 和 [Anand Raman](https://oreil.ly/szRmD) 著

这份资源是我 Azure OpenAI 书籍的完美序言。它包含了 Azure AI 平台的一般信息。这对于你将大型语言模型 (LLMs) 与其他视觉、语言和语音模型结合使用来说是一个很好的补充。

*描述*：这本书展示了云 AI 服务如何与熟悉的软件开发方法相结合，介绍了关键 Microsoft AI 服务，并提供了将不同 Azure AI 服务集成的面向 AI 的架构的实战案例。你只需要具备基本云概念的知识，就可以开始学习。

*为什么选择这本书？* 这本书补充了你在本处所读的内容，超越了 Azure OpenAI 服务，专注于所有 Azure AI 服务。

[*使用 Azure 的云原生基础设施*](https://oreil.ly/ylZJI)，由 [Nishant Singh](https://oreil.ly/MjRMB) 和 [Michael Kehoe](https://oreil.ly/QEOtQ) 著

必须整合 Microsoft Azure 的云原生方面，对于使用 Azure OpenAI 的云原生生成式 AI 解决方案来说，这是非常必要的。

*描述*：通过这本实用的书籍，作者们展示了如何遵循 CNCF 的指南，使用 Microsoft Azure 或其他云计算解决方案构建真正的云原生基础设施。DevOps 和站点可靠性工程师将学习如何在设计阶段早期适应应用程序，这有助于你充分利用云的弹性和分布式特性。

*为什么选择这本书？* 如果你喜欢 第二章，这本书绝对应该排在你的阅读列表中。它探讨了 Azure 的云原生基础、架构和开发周期。更重要的是，如果你喜欢云原生主题，并且想要获得一些官方认证，你还可以访问我的 [*Kubernetes 和云原生认证 (KCNA) 学习指南*](https://oreil.ly/-z6Tt)。

[*在 Microsoft Azure 中实现应用程序交付和负载均衡*](https://oreil.ly/3am-p)，由 [Derek DeJonghe](https://oreil.ly/megO6) 和 [Arlan Nugara](https://oreil.ly/iNdh5) 著

一本关于负载均衡技术的 Azure 首选书籍，对于任何使用 Azure OpenAI 的生成式 AI 应用来说，这是一个越来越相关的领域。

*描述*：随着越来越多的公司将本地应用程序迁移到云端，软件和云解决方案架构师都在忙于研究提高工作负载的负载均衡、性能、安全性和高可用性的方法。这本实用的书籍描述了微软 Azure 的负载均衡选项，并解释了 NGINX 如何有助于全面的解决方案。

*为什么选择这本书？* 这是一本技术指南，专注于非常具体的话题，这些话题将影响你在 Azure OpenAI 上的生成式 AI 开发。在设计你的 AI 着陆区时，网络和负载均衡非常重要，包括结合不同的模型和专用容量（PTU）以及常规按使用付费选项。

[*构建智能云应用程序*](https://oreil.ly/oHr-q)，作者 [John Biggs](https://oreil.ly/S3bup) 和 [Vicente Herrera García](https://oreil.ly/OadSv)

这是对 Azure 中无服务器架构概念的精彩介绍。在了解了 Azure OpenAI 之后，你可能还想探索的其他相关主题。

*描述*：无服务器计算正在彻底改变我们构建和部署应用程序的方式。随着云服务提供商运行服务器和管理机器资源，公司现在可以专注于应用程序的业务逻辑和功能。这本实践手册展示了经验丰富的程序员如何使用微软 Azure 的无服务器架构构建和部署可扩展的机器学习和深度学习模型。

*为什么选择这本书？* 正如你在第二章中看到的，不同的云服务级别对于你的生成式 AI 开发非常重要。这本书是你深入了解无服务器、模型即服务 API 背后的逻辑的另一项技术深入探索。这对于 Azure OpenAI 服务很重要，也适用于 Azure AI Studio 中的其他生成式 AI 模型。

[*生成式深度学习*](https://oreil.ly/11c9N)，作者 [David Foster](https://oreil.ly/1twy1)

从技术角度深入探讨生成式 AI 主题。如果你愿意学习更多关于生成式 AI 基础知识的知识，这本书是很好的延续。

*描述*：本书从深度学习的基础知识开始，逐步深入到前沿架构。通过技巧和窍门，你将了解如何使你的模型更有效地学习并更具创造力。本书还探讨了生成式 AI 的未来以及个人和公司如何积极利用这项令人瞩目的新技术来创造竞争优势。

*为什么选择这本书？* 如果你想要了解生成式 AI 模型背后的技术、科学细节，这本书是必读之选。这是一本非常技术性的书籍，显然对于常规从业者来说不是必需的，但它将帮助你成为一名真正的 AI 专家。

[*生成式 AI 的提示工程*](https://oreil.ly/N75Tj)，作者 [James Phoenix](https://oreil.ly/5m3x-) 和 [Mike Taylor](https://oreil.ly/Z3lmO)

另一次深入探讨，这次是关于提示工程主题。对于不断发展的主题，这是一个极好的资源，并且对于你的生成式 AI 应用高度相关。

*描述*：通过这本书，你将获得生成式 AI 的坚实基础，包括如何在实践中应用这些模型。当最初将 LLM 和扩散模型集成到他们的工作流程中时，大多数开发者都难以从它们中获得足够可靠的成果来用于自动化系统。

*为什么这本书？* 这本提示工程指南是第五章中最佳实践的完美延续。如果你想要继续发展你对基于提示的系统在技术和功能上的理解，这是一本很好的读物。
