# 附录。其他学习资源

现在市面上有大量的资源，但本附录的目标是为您提供一份精心挑选的书籍、代码库、文档和报告列表，以进一步扩展您的通用人工智能知识。

# 适合您技能提升之旅的相关 O'Reilly 书籍

这一系列的书籍包括有助于您在公司层面发展的阅读材料，同时也支持您的一般 Microsoft Azure 技能提升，特别是在人工智能和生成式人工智能技术方面。所有这些都与您在这里学到的知识相辅相成，您还可以关注作者的简介，继续从这些知名专业人士那里学习。让我们来探索它们：

*《人工智能组织》* [David Carmona](https://oreil.ly/FOSlv)

这是我个人最喜欢的一本书。它包含了人工智能采用公司的真实经验，以及从人工智能战略角度进行的专家访谈和经验教训。David Carmona 是高级人工智能主题的绝对参考（以及微软同事）。

*描述*：这本实用指南解释了商业和技术领导者如何拥抱这一新型组织。基于真实的客户经验，微软的 David Carmona 涵盖了成为人工智能组织所需的旅程——从今天在您的业务中应用人工智能到能够使您的组织重新定义行业的深度转型。

*为什么选择这本书？* 人工智能采纳的学习内容包括文化、技术、责任和商业话题，这些对于任何利用人工智能和生成式人工智能技术的公司都非常相关。坦白说，这本书是一部现代经典，也是各种职业人士，包括技术和商业人士都容易阅读的资源。

[*人工智能阶梯*](https://oreil.ly/7bmtW) [Rob Thomas](https://oreil.ly/Ipe6t) 和 [Paul Zikopoulos](https://oreil.ly/1gCVc)

另一本关于人工智能采纳主题的 O'Reilly 经典著作，其中包含其他采用公司的真实案例。

*描述*：IBM 的 Rob Thomas 和 Paul Zikopoulos 向 C 级高管和商业专业人士介绍了人工智能阶梯，这是一种统一、指导性的方法，帮助他们理解和加速人工智能之旅。书中包含真实世界的案例和真实经验，探讨了人工智能的驱动因素、价值和机会，以及组织在采用人工智能时面临的挑战。

*为什么选择这本书？* 与《人工智能组织》类似，这本书包含了人工智能采纳的真实场景。但我最喜欢的是逐步的数据准备，这对于任何想要训练自己的 AI 模型或实施 RAG 或微调模式的人来说都是高度必要的。

[*设计机器学习系统*](https://oreil.ly/7UWAL) [Chip Huyen](https://oreil.ly/AFnId)

在人工智能主题中，最相关的作者之一。技术细节详尽且易于理解。

*描述*：在这本书中，您将学习一种全面的方法来设计可靠、可扩展、可维护且能够适应不断变化的环境和业务需求的机器学习系统。

*为什么选择这本书？* Chip 创建了一个极好的资源，对于任何想要了解机器学习系统端到端操作的人来说，这是一本绝对参考书——评论也一致认为如此。适用于生成式 AI 和 LLMOps，如果您想要一些技术知识和全面的视野，我强烈推荐这本书。

[*《高风险应用中的机器学习》*](https://oreil.ly/acZdT)，由 [Patrick Hall](https://oreil.ly/b-WAX)、[James Curtis](https://oreil.ly/Jexy2) 和 [Parul Pandey](https://oreil.ly/CXpVc) 著

从高度应用的角度对负责任的人工智能主题进行的一次出色介绍。

*描述*：本书描述了负责任的人工智能方法，这是一个全面框架，旨在通过风险管理、网络安全、数据隐私和应用社会科学的最佳实践来改进 AI/ML 技术、业务流程和文化能力。

*为什么选择这本书？* 我们在 第五章 和与 Sarah Bird 的访谈中涵盖了负责任的人工智能（RAI）主题。但 RAI 是一个非常重要的主题，对于那些想要探索 GenAI 技术的潜力和考虑因素的人来说，它值得进行深入的探讨。

[*《机器学习设计模式》*](https://oreil.ly/xdbw_)，由 [Valliappa Lakshmanan](https://oreil.ly/UC8s5)、[Sara Robinson](https://oreil.ly/FoRMy) 和 [Michael Munn](https://oreil.ly/0I6en) 著。

机器学习模型、相关 MLOps、可扩展性考虑等方面的介绍。

*描述*：在这本书中，您将找到关于 30 种数据与问题表示、操作化、可重复性、可再现性、灵活性、可解释性和公平性的模式的详细解释。每种模式都包括对问题的描述、多种潜在解决方案以及针对您所处情况选择最佳技术的建议。

*为什么选择这本书？* 这是一本详尽的、生成式 AI 之前的指南，它高度关注模型训练模式，是一个很好的补充或替代品，可以与 *Designing Machine Learning Systems* 相结合。对于有技术背景的专业人士来说，这是一项额外的学习资源，强烈推荐。

[解码数据架构](https://oreil.ly/sqpJr)，由 [James Serra](https://oreil.ly/3CDHj) 著

一本关于从经典到湖仓架构的架构模式的关键资源，这些模式可以为您的生成式 AI 实现提供支持。

*描述*：数据编织、数据湖仓和数据网格最近作为现代数据仓库的可行替代品出现。这些新架构具有显著的优势，但它们也围绕着很多夸张和混淆。这本书提供了对每个架构的导游，以帮助数据专业人士了解它们的优缺点。

*为什么选择这本书？* 詹姆斯·塞拉（James Serra），微软最有知识架构师之一，就足以成为任何人都应该阅读和推荐这本书的理由。这是一本更新后的指南，展示了现代公司数据平台所需的一切，包括为您的生成式 AI 发展提供动力的湖屋模式。

[*大规模数据管理*](https://oreil.ly/dV1Hk)，作者 [Piethein Strengholt](https://oreil.ly/EJRyo)

是之前书籍的完美补充，特别关注联邦数据系统。

*描述*：随着数据管理不断快速发展，将所有数据集中在一个地方，如数据仓库，已不再可扩展。今天的世界是关于快速将数据转化为价值。这需要我们在联邦责任、数据管理和使其对他人可用的方式上实现范式转变。通过这本实用的书籍，您将学习如何设计一个下一代数据架构，考虑到您组织所需的规模。

*为什么选择这本书？* 与詹姆斯·塞拉一样，皮特海因（Piethein）是这个领域的顶级声音，也是数据管理主题的一个巨大知识来源。这是与 CluedIn 的 Tim Ward 讨论的延续，对于任何企业级实施都是高度必要的。另一本必读之作。

[*学习 Microsoft Azure*](https://oreil.ly/G2U08)，作者 [Jonah Carrio Andersson](https://oreil.ly/gZGyw)

来自社区 MVP 的 Microsoft Azure 云的精彩介绍。如果您想扩展对 Azure 知识的了解，本书强烈推荐。

*描述*：如果您的组织计划将服务现代化并从遗留软件或本地私有云迁移到云，这本书就是为您准备的。软件开发人员、解决方案架构师、云工程师以及任何对云技术感兴趣的人都将学习使用 Microsoft Azure 进行云计算、迁移、转型和开发的基本概念。

*为什么选择这本书？* 对于我来说，这是学习“所有 Azure 事物”的完美方式，如果您缺少核心基础设施、网络、管理、安全和其他主题的基础知识。乔纳（Jonah）也是这本书的技术审稿人之一。

[*Azure 食谱*](https://oreil.ly/lOREI)，作者 [Reza Salehi](https://oreil.ly/W3lgM)

一本包含逐步解释的技术指南，是之前 Azure 书籍的完美补充。这是扩展您对 Azure OpenAI 能力之外知识的绝佳方式。

*描述*：您如何处理在使用 Azure 时遇到的问题？这本实用指南提供了超过 75 个食谱，帮助您在日常场景中处理常见的 Azure 问题。这包括设置存储账户权限、使用 Cosmos DB API、管理 Azure 基于角色的访问控制、使用 Azure 策略管理您的 Azure 订阅等关键任务。

*为何选择这本书？* 对于我来说，这是在 Jonah 的书籍之后的任何 Azure 学习者的第二步。Reza 创建了一个广泛的食谱，对于任何想要利用 Azure 资源的专业人士来说都很有用。部署和管理步骤适用于 Azure OpenAI 和其他人工智能资源。

[*Azure 人工智能服务在云、移动和边缘的规模应用*](https://oreil.ly/UbFc9)，作者：[Simon Bisson](https://oreil.ly/xQwY2)、[Mary Branscombe](https://oreil.ly/JjlGg)、[Chris Hoder](https://oreil.ly/cR_Xk) 和 [Anand Raman](https://oreil.ly/szRmD)

这份资源是我 Azure OpenAI 书籍的完美序言。它包含了 Azure 人工智能平台的一般信息。这对于您将大型语言模型（LLMs）与其他视觉、语言和语音模型结合使用来说是一个很好的补充。

*描述*：本书向您展示如何将云人工智能服务与熟悉的软件开发方法相结合，带您了解关键微软人工智能服务，并提供将不同 Azure 人工智能服务集成的面向人工智能架构的实战案例。您只需具备基本云概念的实际知识即可开始学习。

*为何选择这本书？* 本书补充了您在这里所读到的内容，超越了 Azure OpenAI 服务，并专注于所有 Azure 人工智能服务。

[*使用 Azure 的云原生基础设施*](https://oreil.ly/ylZJI)，作者：[Nishant Singh](https://oreil.ly/MjRMB) 和 [Michael Kehoe](https://oreil.ly/QEOtQ)

对于您来说，整合 Microsoft Azure 的云原生方面是必不可少的，对于使用 Azure OpenAI 的云原生生成式人工智能解决方案来说也非常必要。

*描述*：通过这本实用的书籍，作者向您展示如何遵循 CNCF 的指南，使用 Microsoft Azure 或其他云计算解决方案构建真正的云原生基础设施。DevOps 和站点可靠性工程师将学习如何在设计阶段早期适应应用程序如何帮助您充分利用云的弹性和分布式特性。

*为何选择这本书？* 如果您喜欢第二章，这本书绝对应该排在您的列表中。它探讨了使用 Azure 的云原生基础、架构和开发生命周期。更重要的是，如果您喜欢云原生主题并且想要获得一些官方认证，您还可以访问我的[*Kubernetes 和云原生副协会(KCNA)学习指南*](https://oreil.ly/-z6Tt)。

[*在 Microsoft Azure 中的应用交付和负载均衡*](https://oreil.ly/3am-p)，作者：[Derek DeJonghe](https://oreil.ly/megO6) 和 [Arlan Nugara](https://oreil.ly/iNdh5)

专注于与负载均衡相关的技术主题的 Azure 首本书籍，这对于任何使用 Azure OpenAI 的生成式人工智能应用来说越来越相关。

*描述*：随着越来越多的公司将本地应用程序迁移到云端，软件和云解决方案架构师都在忙于研究提高负载均衡、性能、安全性和高可用性的方法。这本书实用地描述了 Microsoft Azure 的负载均衡选项，并解释了 NGINX 如何有助于综合解决方案。

*为什么选择这本书？* 这是一本技术指南，专注于非常具体的话题，这些话题将影响你在 Azure OpenAI 上的生成式 AI 开发。在设计你的 AI 着陆区时，包括结合不同的模型和专用容量（PTU）以及常规按量付费选项，网络和负载均衡非常相关。

[*构建智能云应用程序*](https://oreil.ly/oHr-q)，由[John Biggs](https://oreil.ly/S3bup)和[Vicente Herrera García](https://oreil.ly/OadSv)著

Azure 中无服务器架构概念的精彩介绍。在探索 Azure OpenAI 之后，你可能还想深入了解的另一个相关主题。

*描述*：无服务器计算正在彻底改变我们构建和部署应用程序的方式。随着云服务提供商运行服务器和管理机器资源，公司现在可以专注于应用程序的业务逻辑和功能。这本书通过实践指导经验丰富的程序员如何使用 Microsoft Azure 的无服务器架构构建和部署可扩展的机器学习和深度学习模型。

*为什么选择这本书？* 正如你在第二章中看到的，不同的云服务级别对于你的生成式 AI 开发非常重要。这本书是另一个技术深入探讨，帮助你理解无服务器、模型即服务 API 背后的逻辑。这对于 Azure OpenAI 服务很重要，对于 Azure AI Studio 中的其他生成式 AI 模型也是如此。

[*生成式深度学习*](https://oreil.ly/11c9N)，由[David Foster](https://oreil.ly/1twy1)著

从技术角度深入探讨生成式 AI 主题。如果你愿意深入了解生成式 AI 基础知识，这本书是一个很好的延续。

*描述*：本书从深度学习的基础知识开始，逐步深入到前沿架构。通过技巧和窍门，你将了解如何使你的模型更有效地学习并更具创造力。本书还探讨了生成式 AI 的未来，以及个人和公司如何积极利用这项令人瞩目的新技术来创造竞争优势。

*为什么选择这本书？* 如果你想要了解生成式 AI 模型背后的技术、科学细节，这本书是首选。这是一本非常技术性的书籍，显然对于常规从业者来说不是必需的，但它将帮助你成为一名真正的 AI 专家。

[*生成式 AI 的提示工程*](https://oreil.ly/N75Tj)，由[James Phoenix](https://oreil.ly/5m3x-)和[Mike Taylor](https://oreil.ly/Z3lmO)著

另一次深入探讨，这次是关于提示工程主题。这是一个不断发展的主题的优秀来源，并且对你的生成式 AI 应用高度相关。

*描述*：通过这本书，你将获得生成式 AI 的坚实基础，包括如何在实践中应用这些模型。当最初将 LLMs 和扩散模型集成到他们的工作流程中时，大多数开发者都难以从它们中获得足够可靠的成果，用于自动化系统。

*为什么选择这本书？* 这个提示工程指南是第五章中最佳实践的完美延续。如果你想要继续发展你对基于提示的系统在技术和功能上的理解，这是一本很好的读物。
