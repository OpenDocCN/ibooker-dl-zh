# 前言

我花了很多时间帮助开发者学习如何将 AI 工具作为他们日常工作的组成部分——通过指导人们、举办培训课程，并在书中撰写关于它们的内容——同时我自己在构建真实软件时也依赖了这些工具。其中一个特别突出：GitHub Copilot。

这是一个颠覆性的变化。我一直在使用它。

我已经编码了几十年，就像每个开发者一样，有时我会遇到瓶颈。可能是在使用新技术或 API 时，我害怕去阅读那些写得不好的文档。可能是在重构团队中每个人都避免的特别棘手的代码时。或者可能是我根本不想编写那些必须编写但会浪费我半小时生命、永远无法挽回的枯燥样板代码。

这就是 Copilot 真正能发挥作用的地方。它可能会给我所有需要的代码，完整且可用。或者它可能会给我一个起点——我们可以一起迭代的东西。无论如何，对我来说都是有价值的，而且几乎总是能让我重新开始。

但与 Copilot 合作并不是一条单行道——它伴随着责任。Copilot 可以编写可编译、可运行甚至通过测试的代码。但如果你不理解它给你提供了什么，或者为什么它有效——或者，最重要的是，你是如何让它产生那个结果的——你实际上并没有控制好你的代码库。这本书让你掌握控制权。

优秀的开发者需要做的不仅仅是学习 Copilot 的功能。我们需要学习一种有效的工作方式，一种支持我们在现有工作中使用 AI 的方式：编写代码、做决策、与其他人一起解决问题。

学习的开始是理解 Copilot 并不完美——它也不试图完美。它试图通过根据你的代码中的模式和上下文提供建议来帮助你前进。很多时候，它都能令人满意地接近完美。但像任何工具一样，当你知道如何引导它，以及何时放慢速度并仔细检查它给你提供的内容时，它才能发挥最佳效果。

正因如此，这本书才有意义。《学习 GitHub Copilot》是实用的、深思熟虑的，并且基于开发者每天如何编写代码。布伦特·拉斯特做得非常出色，因为他不仅仅列出了 Copilot 的功能；他展示了你在实际工作中如何使用它们——当事情混乱、非线性且经常有截止日期时。

他从 Copilot 内置的优势开始，比如解释代码和编写测试，然后展示它是如何帮助人们解决难题的——SQL 查询、正则表达式、YAML 文件，以及其他所有消耗时间的（有时甚至不是那么小的）任务。我非常欣赏书中所有的例子，因为它们反映了开发者们在现实生活中遇到的问题。我想你们也会喜欢的。

Copilot 作为一款重要的日常开发工具，并非因为它承诺了魔法，而是因为它尊重开发者的思维方式。这本书也做了同样的事情。它假设你是深思熟虑的、有能力的，并且仍在学习，它为你提供了使用工具做出良好决策的空间，而不是仅仅跟随他人。

在我自己的教学和写作中，我看到了开发者们在获得清晰、诚实的指导时如何迅速提高——尤其是在他们进行实际工作时，这有助于他们做出更好的决策。这正是这本书提供的。这不是炒作。它不是过于简化。这是一本为希望保持对其代码控制权的开发者编写的实用指南。

如果你刚开始使用 Copilot，这本书是一个很好的起点。如果你已经使用了一段时间，你将学会使它更有效和更容易信任的技术。如果你一直持怀疑态度，这也是健康的。这本书会与你站在同一位置，并展示你如何充分利用这个工具，同时不放弃使你成为一名优秀开发者的重要因素。

安德鲁·斯特尔曼
