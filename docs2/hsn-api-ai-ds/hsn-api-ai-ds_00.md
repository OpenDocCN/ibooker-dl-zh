# 前言

要在 AI 领域取得成功，首先掌握 API。由于人工智能、机器学习和数据科学的增长，精通 API 比以往任何时候都更有价值。

但是学习这样广泛的技术可能会让人感到害怕。这要如何完成呢？你可以从这样一个事实中得到安慰，那就是你不必学习每个技能，当然也不必一次性学习所有技能。通过动手实践一次学习一个技能。你学习的每个技能都会使下一个技能更容易，就像积木一样。

# 为什么你应该阅读这本书？

如果你正在阅读这本书，你想要提升你的技能。我发现通过动手编码是做到这一点的最好方法。如果你通过在公共仓库中发布你的代码、写博客和分享你创建的内容来公开你的编码，你可以传递你的知识来帮助他人。你还将建立一个坚实的作品集，为雇主提供你专业能力的具体证明。

# 这本书适合谁

由于这本书位于 API、AI 和数据科学的交汇点，它将对几种类型的读者都有价值。

## 数据科学家

数据科学家经常使用 API，因此有一种诱惑认为关于使用 API 没有什么新东西可以学习。调用 API 不就是几行代码吗？确实，对 REST API 进行一次调用是一个简单的任务，这也是它们变得如此普遍的原因之一。但是，以稳健和有弹性的方式使用 API——并且不会给提供商带来问题——需要更多的关注。

本书将教授一些你可能还没有学过的技术，例如：

+   开发和部署 API

+   创建软件开发工具包（SDK）和 API 客户端

+   创建并发布 Python 包到 PyPI

+   将机器学习模型作为 API 发布

+   创建 Streamlit 数据应用

+   使用 Airflow 创建数据管道

+   使用 LangChain 和 ChatGPT 创建生成式 AI 应用

## API 开发者和设计师

API 开发者和设计师可以学习如何为重要的新受众增强他们的 API。他们将了解数据科学家：他们所做的工作，他们执行的任务，以及他们喜欢的 API 功能。他们还将了解生成式 AI 应用：它们如何调用 API 以及它们在 API 中需要的功能。

实践中的示例将教授各种新技能：

+   使用 FastAPI、SQLAlchemy 和 Pydantic 创建 Python API

+   使用 Docker 容器化 API

+   将 API 部署到云主机

+   创建 Python SDK 并发布到 PyPI

+   使用 LangChain 和 ChatGPT 创建生成式 AI 应用

## 求职者和角色转换者

上述技能在市场上非常有价值，因此学习它们可以帮助你找到你的第一个角色或在数据科学或软件开发中的新角色。本书围绕构建投资组合项目组织，这将为你提供具体的目标去实现，以及你工作的具体证据。

# 使用本书

我建议你不要从头到尾阅读这本书，而是选择你想要学习的技能，并开始学习教授该技能的章节。你可以通过以下方式快速完成：

1.  决定你是想先创建一个 API (第 I 部分)，使用数据科学中的 API (第 II 部分)，还是使用 AI 中的 API (第 III 部分)。

1.  按照该部分的入门章节中的说明执行操作，克隆 GitHub 仓库并启动 Codespace。

1.  按照相关章节中的说明执行代码。

如果我做得恰当，每一章都可以独立存在。在你学会一项技能后，四处看看并找到你想要学习的另一项技能，然后重复同样的过程。本书中的技能就像构建块：你学习的每一部分都为你更深入地理解其他部分做好准备。所有这些加在一起，给你提供了对数据科学和 AI 中 API 的相当深入的理解。

# 本书不是什么

本书不会教初学者 Python 语法。如果你对 Python 有一定的基础知识，你将能从代码示例中获得最大收益。尽管你可能会按照每个章节中的步骤让代码运行起来，但我建议你从 O'Reilly 出版的优秀的 Python 入门书籍之一开始，例如 Bill Lubanovic 的《Introducing Python, 3rd Edition》。

本书还假设你对在 Linux 或 Unix 中使用命令行终端有基本的了解。你不需要成为 Linux 管理员，但你应该熟悉作为开发者如何在终端中运行命令。（所有步骤都有解释，但当你遇到问题时，如果没有一些该环境的基础知识，你可能会感到沮丧。）

本书介绍了几个有用的 Python 框架，如 FastAPI、Pydantic、Streamlit、Airflow 和 LangChain。然而，它没有涉及在*生产*环境中运行它们的必要详细主题，例如安全性、性能和基础设施。我希望你会喜欢本书中的项目，以至于你将继续使用我在每一章末尾提到的参考继续学习。

请记住，本书中的服务和工具正在迅速变化，因此根据你阅读的时间，一些步骤和图表可能看起来略有不同。

如乔伊·格雷科所说，幻想体育是这种场景的自然选择。（你将在本书的后面听到更多关于他的内容。）幻想体育是数据科学家的自然游乐场，幻想网站使用了大量的 API。多年来，我作为一个虔诚的（或上瘾的）幻想经理，在这两方面投入了大量的时间。幻想经理也是任何幻想网站提供的预测或规范性分析功能的快速采用者。（如果你对此表示怀疑，你没有看过一个经理挑选两名自由球员并做出三次阵容调整，将他们的获胜概率从 45%提高到 53%。）

幻想体育是一种有趣的方式来探索人工智能、数据科学和 API 之间的重叠。当你通过编写本书的方式前进时，我希望你和我写作本书时一样享受乐趣。
