7.1 Creating a contemporary design

Site design fashions change over time, much like architecture, art, and other creative endeavors. One gauge of current fashion is how today’s buildings look. And you can sum up much of today’s architecture with one word: stark.

Western styles throughout history have moved between two extremes: curved (decorative, dynamic, ornate) and linear (rectangular, static, minimalist). We’re currently in a linear period in many of the arts, including the art of website design.

For centuries during the classical age, buildings were characterized by utilitarian simplicity and relatively little ornamentation. Examples include early Egyptian, Persian, and Greco-Roman structures (think Parthenon). For the most part, these buildings feature highly systematic, orderly designs, focusing on proportion, balance, symmetry, and function.

But during a long period from the early Middle Ages up until the mid-20th century, the pendulum swung back the other way, and then got stuck decidedly in a lengthy ornate period. Ornamentation, detail, and complexity reigned for centuries. This lasted through the Gothic, Baroque, Rococo, Victorian, and even the Art Nouveau and Art Deco periods going into the early 20th century. For centuries, architects favored intricate details and extensive decorative elements. From embellishments such as gargoyles and colorful frescoes and stained-glass windows, to elaborate, even fussy, decorations.

Then, during World War II, classical styles returned—this time with severe functionality. Even curves went out of fashion. That’s a simplification, but there’s some truth in it. For the past eight decades, the emphasis has been on linear, unadorned, mainly rectangular shapes. The Sidney Opera House and the curvilinear wonders of architect Frank Gehry are among the few exceptions.

7.2 Understanding CSS styling

Cascading style sheets (CSS) is a specialized computer language that can modify most elements of web page design. So, if you want to fine-tune your theme’s default design choices, you need to use CSS.

Best of all, the YellowPencil plugin automatically writes the CSS code to make your changes permanent in your website.

We’ll explore the excellent YellowPencil design system shortly in section 7.4. Skip down to it now if you want. This next section is a bit technical, and if you prefer not to read it, no harm done. But if you’re curious, here’s a brief tour of what CSS does and how it works. This book is designed to be of value to beginners, but also of interest to us nerds.

Lowercase always

Always use lowercase letters when writing CSS and HTML code. Originally, CSS was wisely case-insensitive. But committees got together, and the result was that CSS was made case-sensitive, unfortunately. So, there are now exceptions to case-insensitivity in both HTML and CSS. The code is case-sensitive in several contexts (class names, attribute selectors), but it’s frequent enough to cause countless bugs. So, to avoid problems, always use lowercase rather than trying to memorize the random exceptions to the rule.

Here’s an example of CSS style definitions for a CSS class name myId or myid:

#myId {
    color: red;
}
#myid {
    color: blue;
}

Here’s the HTML:

<div id="myId">Some text.</div>

Because of case-sensitivity, the text inside these div tags is displayed in red, not blue. Why? Because the “i” in myid is lowercase, and it’s uppercase in myId. So, CSS sees them as two different CSS selectors.

Definition A selector is just the name of the design element that CSS is modifying. The selector is followed by a list of one or more property/value pairs. That entire grouping under each selector is called a CSS rule. Often, the selector is just an ordinary HTML element such as h1. Or it can be a class name such as myid in the previous example.

7.2.1 Exploring the role that HTML plays in page design

Before CSS came along, there was only HTML to tell a browser how a web page should look. But HTML is primarily a markup language because it marks the beginning and ending of a particular element in a web page, such as a heading or a paragraph:

<h1>MAJOR HEADLINE</h1>

That <h1> HTML start “tag” tells the browser to start here applying the h1 style to this text. And the </h1> end tag tells the browser to end that heading style here.

Browsers have a set of default HTML property values. For example, if the browser sees some text surrounded by those h1 tags, it uses the following list of property values by default when displaying the heading:

  • display: block;—This means that an h1 heading starts on a new line and takes up the full width available (that’s what the CSS value called block does).

  • font-size: 2em;—The font size of the h1 element is twice the size of the root element’s font size (aka em).

  • margin-top: 0.67em;—The space above the h1 element is 0.67 times the size of the root element’s font size.

  • margin-bottom: 0.67em;—The space below the h1 element is 0.67 times the size of the root element’s font size.

  • margin-left: 0;—There is no space to the left of the h1 element.

  • margin-right: 0;—There is no space to the right of the h1 element.

  • font-weight: bold;—The text of the headings is boldface.

In other words, all h1 headings in a site are rendered using these default design properties. These bulleted items are what CSS calls property-value pairs, also known as declarations. The whole point is that you can use CSS to modify any of those default values.

7.2.2 Looking at what CSS brings to HTML

What if you wanted to change the size, font, color or other properties of the h1 headings on your site? In other words, you wanted to modify those default browser declarations. Maybe you want to make these headings larger or something. Before CSS, you would have had to go through all the site’s HTML code and change each h1 element by hand. (In HTML, a complete tag-content-end tag unit is called an element.) To make the h1 headings larger, you would have to retype the HTML start tag to read like this:

<h1><font size="+2">MAJOR HEADLINE</font></h1>

This says make the h1 headings two sizes larger than the browser’s default size. Typing in this change to every h1 heading throughout a site was obviously both cumbersome and tedious.

Warning Be sure that any quotation marks are straight (" "), not curly (“ ”), in your CSS code. HTML and some types of CSS code don’t like slanted or curly open and close quotation marks.

Here are some typical HTML tags:

  • <h1> to <h6> define headings, with <h1> being the most important and <h6> the least (usually the smallest). These give a hierarchical structure to the text content on your site, letting readers know the relative importance of, or the relationship between, sections in the page.

  • <p> represents paragraphs (common body text).

  • <div> and <span> specify a zone or a group of page components.

  • <table> is used for tables.

  • <img> is used for images.

  • <a> is used for hyperlinks.

7.2.3 Enter CSS

Fortunately, CSS was invented to make site design far easier and more varied. Among other helpful features, you can change all the h1 headings in a site by modifying just a single CSS rule. (We did this in the fade-in CSS example earlier.) This next example shows a rule that makes all h1 headings twice the size of the browser’s default (2 em) font size):

h1 {
    font-size: 4em;
}

In most modern browsers, the size of the paragraph <p> body text elements defaults to 16 pixels [px]).

Tip The pixel is the smallest visual unit in a display such as a TV or cellphone. If you get up real close to a TV, you can usually see the pixels, that is, little dots. You can’t see them on a cellphone screen because they are too small, but on a jumbotron display in a stadium, its pixels are the size of bricks. So, the size of a pixel is relative to the size of the screen that it’s on. A pixel doesn’t have a specific, absolute size like an inch or centimeter.

CSS also offers many more style properties than were available in the original HTML set of attributes (attribute is HTML’s name for styling modifications). And CSS even goes beyond simple styling. Some prefer to call CSS a markup language, but it’s actually a design language with significant capabilities in addition to markup (merely specifying the start and end of a style). For example, CSS includes the following:

  • Transforms—2D or 3D transformations applied to an HTML element, allowing you to rotate, scale, move, skew, reposition, and make other changes to the elements.

  • Transitions—Morphing an element from one state to another. In other words, CSS transitions allow you to change property values smoothly, over a given duration, at a given speed, and starting after a given delay. This technique can be used to create a variety of effects and animations such as fading in or out, scaling, skewing, changing colors, and rotating an element over time.

  • Layout and positioning—Flex is a model for positioning and aligning elements within a container. CSS Flex specifies how an element grows or shrinks to fit the space available within its Flex container. Three properties—flex-grow, flex-shrink, and flex-basis—work together, giving you an efficient way to align, lay out, and distribute space among elements within a Flex container.

  • Shadowing—Text or frame shadow effects.

  • Special text specifications—Specs such as adjusting line height.

7.2.4 What is the cascade?

This colorful term cascade is pretty accurate. CSS is called cascading style sheets (plural) because there can be more than one CSS style sheet. There are four main style sheets (files that hold CSS code), and they, working together, create the cascade.

This means there can be duplicated CSS properties in the sheets. Let’s say that one style sheet specifies that h1 headings should be red, but another style sheet says they must be green. What’s a poor browser to do? The answer is that there is an order (the “cascade”) to style sheets. The browser must use the color mentioned in the lowest style sheet (or the lowest mention within a single style sheet).

The cascade is confusing at first. Here’s how to visualize this cascade process. Let’s consider two CSS h1 selectors that are in the same style sheet. One of them is below the other in the style sheet, thus it’s the lowest one. This CSS code results in blue h1 headings because the blue value is listed lower than red:

h1 {
    color: red;
}
h1 {
    color: blue;
}

Put another way, if a CSS property repeats, then the property value (such as the color) lowest in the code is used, and any other previous duplicate property definition (higher in the code) is just ignored. Those previous values are said to be overruled. Just the way that a school principal can overrule a teacher.

To better understand what “lower” means in this context, it helps to visualize a cascade of water, falling down a hill. When deciding which duplicated declaration to use, the browser looks at all four style sheets in a specific order. This is the cascade, as described here:

  • There are four main outcroppings in the CSS cascade, and at each one, if any CSS selector’s property-value pair declaration is repeated, then any previous such declarations are overruled.

  • Visualize a water cascade with four drops or outcroppings. A basket of flowers is placed in the water at the highest outcropping. The flowers are arranged in a particular order. But then the basket falls down to the second outcropping pool, and the flowers might be rearranged by that bump. Then, it bumps down the third and fourth outcroppings, and comes to rest in the lowest pool. If a CSS declaration (property-value pair for a particular selector) is redefined during any of these bumps, that new redefinition is the one used, the one that “wins.” In other words, each lower outcropping in the cascade can redefine (overrule and change) any CSS style that was previously defined by an earlier outcropping.

  • If outcropping 3 says h1 is red, but outcropping 4 says green—green wins and the browser follows that rule when displaying the h1 headings on the site. Each lower outcropping can redefine any selector’s property value, so the lowest outcropping that mentions the color of the h1 heading is the color that the browser ends up displaying. Figure 7.4 An illustration of our flower basket analogy. Each bounce down the cascade potentially rearranges the flowers in the basket.

7.2.5 The order of the style sheets in the cascade

As mentioned earlier, there are four main sheets in CSS. Here is the cascade of sheets in order, starting from the highest to the lowest style sheets:

   1.  The first, highest outcropping is the browser’s default styles. All other subsequent, lower outcroppings can overrule these default style definitions. For example, browsers default to the color black for h1 headings. This is the baseline, and if no subsequent style sheets later in the cascade mention h1’s color, then h1 is rendered black by the browser. (And this is pretty common given that black on white text is usual on sites.)

   2.  Each WordPress website has a theme, and each theme has its own CSS style sheet. It’s the cascade’s second outcropping. You can look at it in your WordPress backend by going to the Appearance > Theme File Editor screen. The theme’s style sheet overrules the browser defaults (but can be overruled by the next two outcropping drops).

Warning You can look at the theme’s style sheet if you wish, but don’t modify it or any of the theme’s other code files.

   3.  Your (the site administrator’s) CSS style specifications come next. In your Appearance > Customize > Additional CSS screen, you can type in CSS code that overrules the previous two style sheets (browser’s defaults and theme’s). This is the third outcropping.

   4.  Finally, at the lowest level, is what’s called inline CSS. It overrules those previous three style sheets. Inline CSS is different from the other levels. Those first three style sheets are global—their style definitions are applied to all elements (like all the h1 headings) throughout the entire site.

However, inline CSS is local and thus defines styles for only a particular zone. That zone can be as small as a single character. Or it could be a word, sentence, paragraph, entire page, heading, image, or some other local content area. One way to specify the zone is to enclose something between <span> </span> or <div> </div> tags, like this:

<span style="color: red;">This sentence is red</span>

In the classic WordPress editor, inline CSS code goes into a Text tab on the page’s Edit Page screen. Or for the Gutenberg block editor (when using a Paragraph block or similar block), click the three dots (Options) in the upper-right corner of an Edit Page screen. Then, click Code Editor, or press Ctrl-Shift-Alt-M.

Here’s the inline CSS code that makes a single sentence red:

<p style="color: red;">This is a red sentence.</p>

This part is the actual CSS code:

style="color: red;"

Rules and selectors

Recall that a CSS rule is the entire list of properties and their values for a particular HTML element. The rule starts with what’s called the selector (that’s what CSS calls the HTML element being styled). The selector is followed by an open curly brace ({) and the list of property:value pairs. Finally, the rule ends with a close curly brace (}). Here’s a typical CSS rule:

h1 {
    color: blue;
    font-family: sans-serif;
    font-size: 14em;
    font-weight: bold;
}

The Inspector also lets you live-edit the CSS code. Any changes made in the Styles pane are immediately reflected on the Inspector’s web page pane. (The page was displayed previously with the two code panes, though the panes can be resized and reordered.) This way, you can experiment with different CSS styles and see the results instantly, without needing to edit the source files and refresh the page.

You can edit existing rules, disable rules, add new rules, and even add entirely new style sheets. (To see more of the frontend pane, move your mouse just above Performance on the menu, and then drag down.) Any changes you make in the Inspector aren’t permanent, however. You’re just experimenting, but the changes aren’t applied to any style sheets.

The Inspector also includes various tools for working with more complex CSS features. For example, the box model view shows the padding, border, and margin for the selected element, and allows these values to be edited directly. (To see the box model for the selected element, click the Computed tab above the CSS code.) Below the box illustration, you can see the final result of the cascade as it applies to this element. In other words, you can see which values won out after the cascade is finished. This can be useful for understanding complex style interactions. (Computed here means the final result after the entire cascade has been scanned and rendered.)

The most popular browsers’ Inspectors also include tools for working with CSS animations and transitions. You can view and control the timeline of animations, slow them down, and pause them at specific points. This can be a help when debugging animations. The various browsers’ Inspectors differ a bit in what they can do, so here’s an overview of each:

  • Chrome—Chrome’s Inspector includes an Animations Drawer tab that automatically detects and groups animations. You can then inspect animations, slow down or replay them, and see the source code.

  • Edge—The Animation Inspector in Edge’s Animations tool supports CSS animations and transitions. You can manipulate and test various effects, including changing the timing, delay, duration, or keyframe offsets.

  • Firefox—Firefox’s Page Inspector includes an animations view that displays animations in the page synchronized along a timeline. It supports animations created using CSS transitions or keyframes rules.

  • Safari—Safari’s Web Inspector provides a graphics feature that enables previewing keyframes and their classes from CSS animations and transitions.

The Inspectors also include tools for responsive design testing. You can simulate different screen sizes, resolutions, and even specific devices, allowing you to see how your CSS styles look on various screen’s sizes and shapes.

Simplifying animation with keyframes

A keyframe is an important moment of change (or “beat”) in an animation sequence. A keyframe can specify when something begins to move, changes direction, resizes, and so on. Keyframes can be set for things such as position, rotation, scale, and so on. Between two keyframes (or a keyframe and the end of the sequence), the animation software will automatically interpolate the values to create smooth transitions.

This means you don’t have to manually define every tiny change, just the important keyframes. You can just say, rotate for 1 second, rather than specifying each position for every one of, say, 24 frames per second. You use this shorthand keyframe technique to specify timing, pacing, transformations, and other animation effects.

7.4.2 Understanding CSS animation

Currently, YellowPencil offers the following animation features:

  • Animation—Using this CSS property you animate HTML elements.

  • Animation generator—This tool helps you create your own custom animations. Its primary function is to enable you to create animations in a way that uses the visual strengths of designers, rather than requiring them to write code manually. You can see the results of these animation designs in real time too.

  • Animation manager—Here, you can manage all the animations that you’ve added to your website. For one thing, you can visually adjust settings such as duration and delay time.

  • Transition—This CSS property allows you to add simple transitions to HTML elements, such as fading out or fading in. And specialized tricks like a rubber-band animation.

  • 50+ animations—This is a collection of more than 50 predefined animations.

Working with animations and transitions

Animations and transitions are related. However, animation is more complex and versatile. You can use multiple keyframes and continuously play or loop the activity without requiring user interaction or other triggers. Animation can involve multiple properties and allow fine control over timing and sequencing, displaying more complicated and varied movement and effects.

Transitions, by contrast, are typically used for simple interactions involving a change from one state to another (such as invisible to visible, or black to pink). They are triggered by user interaction (such as viewing a new page) and typically involve a smooth change of one or more CSS properties between two states: the initial state and the final state. An example is changing black normal text to pink italics, and back again. To sum up: the Animation Generator tool is used for creating more complex, keyframe-based animations, while the transitions tool creates simpler, state-based effects.

This list describes the primary transition properties (https://mng.bz/XxAl):

  • Type—Specifies the name of the CSS property the transition effect is for. The transition effect starts when the specified CSS property changes.

  • Duration—How many seconds (s) or milliseconds (ms) a transition effect takes to complete.

  • Easing—The speed curve of the transition effect. The values can be Ease, Linear, Ease-In, Ease-Out, or Ease-In-Out.

This list describes the primary animation properties:

  • Animation—Adds an animation to an element.

  • Trigger—Specifies what triggers the animation to start. It can be Onscreen (it plays when the element is visible on the screen, a value we’ll use shortly), Hover (starts when the mouse pointer is moved over the element), Click (when the element is clicked), or Focus (triggers when a form field is focused, meaning that the user has pressed the Tab key to move around a form’s fields, or clicked a field).

  • Trigger repeat—Determines how many times the animation runs.

  • Duration—Dictates how long an animation should take to complete one cycle.

  • Delay—States how long after being triggered the animation should wait to begin.

  • Easing—Sets the speed pattern. The values are Ease, Linear, Ease-In, Ease-Out, or Ease-In-Out. Experiment with these to see the differences among them.

  • Animation fill mode—Sets the state of the end animation when the animation isn’t running. The values can be None, Forwards, Backwards, or both Forwards and Backwards.

Tip Here’s a good tutorial from MDN Web Docs on the many ways you can animate your pages using CSS: https://mng.bz/yW27.

7.4.3 Try animating

Here are the steps you can take to fade something in. This can be a subtle, but effective, way to display various elements such as your site’s tagline underneath the site title. Visitors will see your site’s title right away, but then your tagline will take a few seconds to fade into view.

In this example, we’ll make the site’s h1 headings take 2 seconds to fade into view using the YellowPencil plugin’s Motion feature:

  1. Go to your Pages > All Pages screen, and then click a page named Practice to open its Edit Page screen.

  2. Click the Edit Page - YellowPencil button in the top-right area.

  3. Click the page’s title (Practice) in YellowPencil to select that h1 element.

  4. Click Motion on the right side to open its module. Click FadeIn as the animation type.

  5. To start the animation when the element becomes visible on the user’s browser, select `onscreen` for the `Trigger` property.

  6. To play the animation only once set the `Trigger Repeat` property to `1`.

  7. Set the `Duration` property to `2` (seconds)—or `3`, or whatever pleases you.

  8. Leave the `Delay` property at its default 0 value unless you want the animation to start sometime after the page displays.

  9. Set the `Easing` property to `ease` or `linear` depending on how you want the speed of the animation to vary over its duration.

  10. Set the `Animation Fill Mode` to `forwards` to ensure that the page title element remains visible after the animation ends.

  11. Click the Save button in the upper-right area to store the code on your website.

  12. After you follow those steps, YellowPencil generates this CSS code:

.hentry .entry-header h1:yp-onscreen{
    animation-duration:2s;
    animation-delay:0s;
    animation-name:fadeIn;
}
/* Post Title */
.hentry .entry-header h1{
    animation-fill-mode:forwards !important;
}

Now return to your website (click the previous browser tab), and click View Page on the top menu bar. You’ll see the page title Practice fade in. To see the effect again, press F5 to refresh the browser page.

7.5 Understanding CSS scope

Here, we’re entering another nerd zone. This next information gets a bit specialized, so if you’re not a techie, feel free to skip the next couple of sections and tune back in at section 7.8.

You might have noticed when looking at the YellowPencil CSS Editor that it has three tabs: Single, Global, and Template. They govern the scope (the range of influence) of the CSS code. The Single tab only applies that code to the current page, whereas the Global tab applies code to the entire site.

The Template tab is similar to Single, but Template applies the CSS to a particular type of page (any page that is governed by a specific template). For example, say you’re editing a product page controlled by an e-commerce plugin such as WooCommerce. If you click the Template tab, the CSS styles you apply to the current page also apply to all the other product pages on your site. You can thus maintain a consistent look and feel across all pages of the same type.

To review, here is the most useful, uncomplicated visualization of the CSS cascade. It’s a four-level process:

  1. The browser default properties that the web browser applies to HTML elements if no CSS styles are defined further down the cascade. This is the highest (earliest) level. These properties are intrinsic to the web browser, and not part of WordPress, your site’s database, or any CSS style sheet.

  2. The site theme’s CSS file (located in your Appearance > Theme File Editor screen). The styles defined in this file are applied site-wide unless they are overruled by duplicate properties specified further down the cascade.

  3. CSS code that you—the site’s designer—put into the Appearance > Additional CSS screen. These properties are also site-wide (unless you’re defining a CSS class, which can be used in specific <div> elements, for example).

  4. CSS code added within the HTML in a single page or to smaller zones within that page (a word, an image, a paragraph). This is the lowest and final level of the cascade and overrules everything previous. For example, you can go to your Pages > All Pages screen, click the Practice page, and then click the Text (HTML code) tab in the text box.

Now add this HTML/CSS code:

<div style="color: purple;">
  This text will be purple.
</div>

The HTML part of this code is

<div>
  This text will be purple.
</div>

And the CSS portion of the code is

style="color: purple;"

The <div> tag allows you to create a zone where you can apply multiple CSS modifications at once. Here, a foreground color is changed to white, against a purple background color:

<div style="color: white; background-color: purple;">
<h2> A new color combination!</h2>
<p> This text will be white on a purple background.
</div>

We wanted white against purple for both this <h2> heading and the <p> body text below it. So, we created a <div> section to contain the two elements.

Note Recall that the four-level cascade described previously is, to be honest, an oversimplification. CSS has evolved since its introduction in 1996 and, as a direct result of several committee meetings, the CSS cascade has sadly become quite a bit more complex. There are now various kinds of inheritance, scoping limits, scoping roots, proximity, and other factors that can be used if you wish. If you’re interested in learning more about this complexity, you can dive into the latest official committee document here: www.w3.org/TR/css-cascade-6/.

It’s considered best practice to limit your use of inline CSS. If overused, it can make your styling less readable and harder to maintain. After all, CSS was originally invented to eliminate the need to hand-code each HTML element of a particular type everywhere on a site.

If you find a need to do quite a bit of inline CSS, consider using CSS classes rather than the style= attribute. We’ll turn our attention to classes next.

7.6 Working with CSS classes

CSS classes serve a purpose similar to what CSS in general offers: classes separate the CSS code from the site’s content. Recall that you can define a rule for h1 headings (green and sans serif, for example) in the Appearance > Customize > Additional CSS screen. When defined in that screen, all the h1 headings on the site are automatically rendered green and sans. Thus, the styling code doesn’t need to be hand-typed into each h1 element on the site. Instead, the styles defined once in the Additional CSS screen are automatically applied to every h1 element.

That’s clearly a more efficient approach than having to type in the same style properties dozens of times using inline styles for each h1 heading on the site. Another benefit is that if you change your mind and want the color to be blue instead of green, you only have to make that change in one place by modifying the h1 CSS rule in the Additional CSS screen. Then, the change will radiate throughout the site, modifying all the h1 headings.

CSS classes can also be defined in the Additional CSS screen. Classes can be used with inline CSS (they can also be used in external style sheets and elsewhere). A CSS class can be added to individual HTML elements such as h1, one by one, with inline CSS. Similar to how you use inline CSS, its style= attribute is typed in to each h1 element on the site. In other words, you type a class attribute into an HTML element just as you did when adding a style attribute for inline CSS.

Inline CSS using the style= attribute is shown here:

<p style="color: green; font-weight: bold;">This
is a green and sans sentence.</p>

And here’s inline CSS using the class= attribute:

<p class="greensans">This is a green and sans sentence.</p>

So, if you have to type in a class attribute to every h1 element that you want to apply the class’s style to, what’s the point of using classes at all instead of the style= attribute? Well, there are two advantages to using class= instead of the inline CSS style=:

  • When a rule has multiple properties, the style= attribute must list them all for each HTML element (each h1 heading, for example). But with the class= attribute, you need only use the class name, such as greensans shown in the previous code. This makes things shorter and simpler.

  • If you later decide to change the color from green to brown, you only need to make that change in the class definition, instead of modifying by hand every h1 heading individually using style=.

Note that there are several kinds of selectors in CSS, but we’re concerned here with the two main ones: HTML element tag names and classes. HTML tag names are predefined, and you can’t change them. You must use h1 when referring to that heading element. By contrast, you can name classes anything you choose—you make up the names for classes. Among other things, this means if you want to overrule a class defined by the developers of your theme’s CSS, you have to figure out what exactly those developers called that class. (This used to be hard to figure out sometimes, but when using the YellowPencil plugin, it’s no problem at all.) Let’s now try an example that shows how to use a class.

   1.  Go to your Appearance > Customize > Additional CSS screen, and type in this class definition (use any name for this class that you like):

.greensans {
  color: green;
  font-family: sans-serif;
}

   2.  Click the Publish button.

   3.  Go to your Pages > All Pages screen, and then click a page title to open its Edit Page screen.

   4.  Click the Text tab on the top-right area of the text box.

   5.  Type the following into that page’s text box:

<h1 class="greensans">My first headline</h1>
<h1 class="greensans">The second headline</h1>
<p class="greensans">You can apply a class to any HTML element, like this
 paragraph.</p>

   6.  Click the Update button.

   7.  Click View Page to see the changes.

Defining classes

Although WordPress doesn’t provide direct access to the entire HTML code for a page or post, classes are sometimes defined within a <style> </style> tag pair. This pair is located at the top of the HTML for a page, inside the <head> </head> tag pair. This zone is called the head of the HTML document, and is where metadata not visible to the user is located—JavaScript, “internal” CSS, and so on.

Classes can also be defined in an external style sheet that’s linked to a page in the <head> section, like this:

</head>
<link rel="stylesheet" href="styles.css">
<body>

Here are the major zones within an HTML document, illustrating where a page-wide CSS class can be defined:

<html>
<head>
<style>
.greensans {
  color: green;
  font-family: sans-serif;
</style>
</head>
<body>
    <!-- The primary content displayed in the web page-->
</body>
</html>

If you define a class in this head location, it only applies to that particular page. However, to type in “internal” CSS in WordPress, you can’t use the code view (the Text tab in an Edit Page screen) option. WordPress doesn’t display all the HTML in a page when you select code view. Instead, you have to edit the theme’s header.php file. And that’s not a good idea unless you’re an expert PHP coder. Even then, when the theme updates, your class definition will be overwritten.

This ends the tech zone. We now return to our regular content.

7.8 Inpainting and outpainting

AI inpainting makes it easy to remove or replace unwanted objects in an image. With inpainting, you select an object by dragging a selection lasso around the object. Then, you either request its removal or describe what you want to replace it with.

Note Many graphics in this book are meant to be viewed in color. The eBook versions display the color graphics, so they should be referred to as you read. To get your free eBook in PDF, ePub, and Kindle formats, go to www.manning.com/books/wordpress-power-toolkit to register your print book.

Outpainting is the opposite of cropping. Recall that when you crop a picture, you cut off one or more of its sides. This is often done to move the focal point onto an intersection of the lines in the rule of thirds. This can also improve the balance of the image.

Now, thanks to AI, you can (amazingly) extend the image, adding content to one or more sides that never existed before! AI invents additional visual content beyond the current view, but it doesn’t duplicate existing content. Instead, it creates a variation, a visually logical extension of the existing content.

If, for example, you’re working with a beach scene, you could extend the beach on the left or right side, or both. Outpainting in this way will retain the qualities of the current beach, but widen the image. Or you could expand the top of the image to extend the sky. You also can extend the foreground by outpainting past the bottom of the image. It’s a new technique and not available prior to AI.

When outpainting, AI doesn’t just copy the existing content. You won’t get a second cloud that looks like a cloud that exists in the original image. Remember that AI is intelligent. It can perceptively interpret what would likely appear in a wider view of a scene and give you variations to choose from. Extend the beach, and you could easily get just more beach, but with variations in the sand, or shadows, or perhaps a palm tree as well (or you could type palm tree into the generative outpainting toolbar if you do want a palm tree in the outpainted space). These are techniques worth knowing, so let’s take a look at how they’re done.

Although Midjourney and other text-to-image systems have tools that can generate these modifications, currently Photoshop’s tools are the easiest to master. So, let’s use Photoshop to do some inpainting.

Photoshop calls inpainting and outpainting generative fill and generative expand. Both will automatically match the lighting, perspective, and other qualities that preserve the overall content and style of the image. This happens in seconds, and the results are often startlingly appropriate. It’s as if the original photographer had used a wider lens, or Dali had painted three elephants on stilts in the background.

Take a look at figure 7.14. This Midjourney-generated golden mask can benefit from some modifications. First, there’s no strong focal point. So, we’ll make the lips the focal point by asking Photoshop to color them with vermillion enamel. Based on what area we select, the shape of the lips can also be changed.

Figure 7.14 Photoshop will generate three variations of red enamel on the mask’s lips, now that they have been selected with the lasso tool, and our prompt described the material we’re after.

In Photoshop, you can use its lasso tool to draw a selection region. This is the area that will be replaced with enamel. So, with the lasso, I roughly followed the outline of the lips, which leaves behind a dashed line showing what’s selected. A generative toolbar appears in the bottom of the screen, as you can see in figure 7.14. If you just click Generate, without typing in any text, it removes the selected object, filling in the space with whatever surrounds it.

However, our goal is to modify, not delete, these lips, so I typed in “glossy vermillion enamel” as the text prompt, and then clicked Generate.

You can repeatedly click Generate, getting three new variations each time, until you find one you like best. Figure 7.15 shows the best result out of nine variations. The coloring is good, and the pursed shape of the lips is more intriguing, at least to me.

Figure 7.15 The best of nine variations, featuring pursed lips for added interest

Now that the lips are vermillion, to make them clearly the focal point, we’ll use the cropping tool to move the lips onto a rule-of-thirds grid line. Also, the eyes are now on two intersections, as is the right side of the face. We’re outpainting now, and, in this case, we’re dragging the frame down and to the right. These currently black areas will be filled in by Photoshop with appropriate variations on the current background, or with a new background that I specify.

For outpainting in Photoshop, you use the crop tool and drag one or more of that tool’s frame handles to show where the expanding should take place.
