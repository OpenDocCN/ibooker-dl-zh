A WordPress plugin is like an extension in Chrome, Edge, or Safari (Firefox calls them add-ons). Plugins are typically small programs that can be installed in WordPress to add functionality. Examples include spam blockers, e-commerce managers, apps that help improve the site’s rank in Google or AI searches, contact form builders, security systems, and many others.

You can find more than 50,000 plugins in the WordPress plugin directory (https://wordpress.org/plugins/). And thousands more plugins are available from third-party websites.

It can be daunting for site owners to find just the right plugins to improve their websites. That’s where this chapter comes in. Here, you’ll find

  • which kinds of plugins are useful in almost every type of site.

  • how to find plugins of the highest quality.

  • how plugins can help you incorporate certain best practices (such as ensuring accessibility).

  • how to create your own plugins and widgets using AI as your programmer.

5.1.2 WP Rocket

Many consider WP Rocket to be the best all-around, full-service performance plugin. Its primary function is to maximize site performance (such as speeding up page display) using various optimizations. Among other strategies, WP Rocket offers efficient caching. That said, you might want to check with your hosting service. Some hosts don’t want you to add caching plugins to your site because they perform caching on their end. Also, if you run into an unexplained bug on your site, try disabling your caching plugin to see if the bug goes away. Caching plugins are one of the main causes of plugin conflicts.

Definition Caching means storing whole pages, so the browser doesn’t have to redraw a page the next time it’s viewed. If cached, a page can just be slapped into the visitor’s browser window, no processing required. In addition to the text or media, a web page includes hidden HTML and CSS codes that describe things such as layout, headline sizes, font color, and so on. Without caching, each page—even if nothing has been changed in it—must be re-rendered before it’s displayed.

In addition to caching web pages, WP Rocket also employs other types of caching. It offers cache preloading (anticipating, then rendering in advance, what page you’re most likely to view next). It also compresses. One type of compression is called code minification—it removes unneeded characters (such as programmer comments and whitespace). For one example, instead of storing five blank lines, it stores only three characters 5bl (this describes the number of blank lines rather than actually storing a mirror image of all the characters in all the lines). This tactic makes code files smaller, so they load more quickly and require less bandwidth to transmit.

Similarly, WP Rocket concatenates your site’s page files—combining multiple code files into a single file. If not concatenated, to render a page, the browser must make multiple individual requests to your site’s database (stored on your hosting service’s drives)—one request for each file of the several that make up the page. If the files are merged, only one request is needed, and the result is faster load times and improved overall site performance. It’s similar to the difference between researching via Google and asking GPT a question. Google gives you a list of several results that you then must look at individually and sometimes merge into a final answer. GPT does that for you: displaying a single, finished answer to your question.

AI plugins and tools

These plugins provide essential services and are the best of breed. Of course there are always exceptions. If you’re running a personal blog for family and friends, you clearly don’t need to gather data via MonsterInsights or attempt to climb high in Google or AI search rankings using AIOSEO.

That said, your site will likely need some specialized plugins to assist with tasks needed by your site’s particular purpose. To help you make the best choices, we’ll next consider three things you should look for when picking the best plugins.

5.2 Installing the best plugins

Given that there are tens of thousands of WordPress plugins, they range across a spectrum of quality and safety—from completely useless or corrupt to excellent and well-maintained. Look for three things when choosing a plugin:

  • Ensure that it’s popular and gets good reviews. If millions of people are using it, clearly word has gotten around that this is a great choice. You can google pluginsname reviews or for a category, use wordpress security plugin reviews. You can also prompt AI:

You’re an experienced WordPress plugin critic. Please list the three most popular spam-blocking plugins, and explain why you selected them.

  • Or you can read user reviews by going to your Plugins > Add New screen and searching for the name of the plugin or category. In figure 5.4 you can see that Wordfence has more than 4 million users and gets a stellar 4.5-star rating.

Figure 5.4 If a plugin has millions of installations, you can feel confident that it’s of high quality and will be around for a while.

Also, see if a plugin is compatible with the current version of WordPress. WordPress is updated very frequently, often for security reasons. You’d be surprised how many plugins and themes are abandoned by their developers and thus no longer kept compatible. This can pose a security risk to your site.

Tip There are exceptions to that keeping-updated rule. Some out-of-date plugins are so useful that they can still be worth trusting. Easy Google Fonts and Advanced Editor Tools both fall into this category. Both haven’t been updated in two years. So why feel safe with them? For one thing lots of people are still actively using them. Advanced Editor Tools has millions of active installations, and Easy Google Fonts has 200,000. So even if they suffered a security breach, it’s unlikely that you would be among the first victims. Further, your Wordfence or other security would respond rapidly to block the threat. And finally, you’ve got site backups if the worst happens. On the other hand, an abandoned plugin with 3,000 installations is pretty risky. In that case, find an alternative.

  • Chose freemium plugins. A popular plugin might have two tiers, a free version and a paid version. Plugins that have a paid version are far less likely to be abandoned by their designers because they’re generating income. And often the free versions can suffice for many sites. Examples of freemium plugins are Wordfence, AIOSEO, Jetpack, Akismet, SeedProd, MonsterInsights, and UpdraftPlus. In each case, their free version can be quite serviceable.

Asking AI for recommendations

As always, AI can assist you in deciding which plugins are best for your site. For example, if you’re looking for an alternative to Wordfence for your general security plugin, try this prompt:

You’re an expert on WordPress security plugins. I know that Wordfence is very popular, but could you list the pros and cons of two alternative plugins that would be as good as Wordfence?

5.4 Avoid adding too many plugins

While plugins can be very effective and add excellent features to your site, using too many can lead to a variety of problems, including security breaches, bugs, crashes, and slower performance. To avoid these problems, it’s recommended that you only install the plugins you really need.

There is no set rule about the ideal number of plugins. You do want to ensure that you add the functionality that’s necessary for the smooth running of the site, and for the best experience for your site’s visitors. But note that plugin conflicts are a common source of bugs. And the more plugins you use, the more you increase the likelihood of these conflicts.

Plugins can also cause security problems. WordPress allows you to install plugins without activating them. But unless you have a plugin that you frequently turn on and off, you should delete any that aren’t activated. Even though a plugin isn’t being used, it’s still a chunk of code in your site’s database. And programming code can be an entrance for malware, especially if it’s not being updated. (WordPress notifies you if an update is available, even for inactive plugins. And the auto-update feature works with both active and inactive plugins as well.)

TIP The difference between installed and active can be confusing. Think of a lamp. Installed means it’s plugged in. Active means it’s turned on.

5.5 Preventing plugin conflicts

When you have something weird happening in your backend—a bug, an oddity—check for plugin conflicts first. WordPress bugs are caused by a variety of sources: theme incompatibility with the core WordPress functionality, hosting service database problems, and other bugs can cause WordPress to misbehave. But most often, the likely cause of a bug is a conflict between incompatible plugins.

Why do plugins cause bugs? Given that there are an estimated 60,000 plus available, clearly no plugin developer can check to see if their plugin is compatible with every other plugin. If something strange and inexplicable is happening in your WordPress site, your first move should be to follow these steps to see if a plugin is causing the problem:

   1.  Back up your site, and then activate your Coming Soon plugin to block visitors from accessing the site. If the problem can’t be quickly solved, then work on a copy of your site (called staging) rather than block your audience. Your hosting service might have a staging utility, or you can use the popular BlogVault plugin BlogVault WordPress Backup Plugin (https://wordpress.org/plugins/blogvault-real-time-backup/).

   2.  Go to your Plugins > Installed Plugins screen.

   3.  Click the Plugin checkbox to select all the plugins at once, as shown in figure 5.9.

Figure 5.9 Click this Plugin checkbox to select all of your plugins.

   4.  Choose Deactivate from the Bulk Actions list, and then click the Apply button shown in figure 5.10.

Figure 5.10 Deactivating all of your plugins

Now you’ve turned all your plugins off. So, return to the bug and see if it’s now fixed. If so, one of your plugins is the culprit. To find out which one, reactivate them one by one. And each time you activate a plugin, return to the bug. When the bug reappears, the plugin you just activated is the culprit. Uninstall it, and then find an alternative that does the same job.

TIP Another tactic is to use the free Health Check & Troubleshooting plugin to test your plugins. It’s created by members of the WordPress open source community to work with the built-in Tools > Site Health feature. Search for, install, and activate that plugin in your Plugins > Add New screen.

5.6 Adding Jetpack to your site

In 2005, WordPress split into two platforms, the paid version (WordPress.com) and the free open source version I recommend and use in this book’s examples (WordPress.org). The open source .org version of WordPress however does require that you use create and maintain it on a paid hosting service. (Choose SiteGround, Bluehost, or DreamHost, as recommended by WordPress itself.)

Note These “com” and “org” names are used only to identify the two versions of WordPress and have nothing to do with domain extensions (such as .com, .net, and so on) that appear at the end of internet addresses.

WordPress.com includes Jetpack, a collection of fairly lightweight plugins (called modules as mentioned earlier), and some of them are quite useful. These modules are lightweight because, compared to many other plugins, they don’t offer lots of additional features. But they do get simple jobs done. For example, the Jetpack Form module displays a bare-bones form that a visitor fills out to subscribe to your site, apply for access to your blog, or access other kinds of features. Among other things, a contact form helps you build a list of email addresses from your audience.

By contrast, a plugin named Contact Form 7 is a full-featured alternative to the pretty basic Jetpack Form module. But if all you want to do is simply harvest email addresses, the Jetpack plugin works fine. The visitor fills in their information, clicks a Submit button, and the form is emailed to you.

Requiring an email address before a visitor can post messages to your blog is one way of cutting down on spam or other inappropriate messages. To turn on this requirement, go to your Settings > Discussion screen, and check Comment Author Must Fill Out Name and Email.

There is a small drawback to using Jetpack though. Before you can activate it, you have to set up a free account on WordPress.com so you can link your WordPress.org site to that free account. Only then can you install the Jetpack plugin itself on your .org site.

This free .com account’s site need not be populated with content or otherwise managed. You can just let it sit there and never return to it once you link to it. The WordPress.com system ignores it too, even if it remains inert. Your reward for setting up that free account is the 46 modules you add to your site with Jetpack. (Some Jetpack modules are activated by default; others require that you activate them by hand.)

The process of signing up with WordPress.com varies. Your goal is to sign up for the free version. If you already have a .com account, you can skip the process of setting up a new account (so skip down to the next paragraph). During the setup process, you need to pay attention to making the correct choices to establish your free account—ensure that you choose the correct word free because sometimes that term is hidden (so scroll down) or several options are displayed using terms like “free to set up” but only one—the truly free one—is unmodified by other words.

5.7 Dealing with the EU cookie law

The European Union cookie law (aka the EU ePrivacy Directive or the Cookie Directive) was introduced in 2011. Its main purpose is to protect the privacy of internet users by requiring website owners to get consent from their visitors before storing cookies on the visitors’ devices. Cookies are small text files saved on a user’s computer and can be used to track their activity on a website, personalize their experience, and collect data for advertising or other purposes. In practice, the enforcement of the EU Cookie Law varies between EU member countries because each country has included the directive differently in its national law.

In 2018, the EU updated the law with a General Data Protection Regulation (GDPR). It strengthened the rules for obtaining consent from site visitors and increased the penalties for noncompliance. Currently, EU lawmakers are cooking up a third update to make further changes to the existing directive. They’re lawmakers. It’s their thing.

Some countries, such as Canada and Australia, have similar laws in place. In the United States, there is currently no federal cookie law, but tireless lawmakers in some states have passed their own laws.

How should you deal with this? If you’re running a personal blog, intended for local consumption only, or if your site is just for your students, friends, or relatives, you likely don’t need to store cookies anyway. But if your site does use cookies, it would be polite to let your visitors know what information you’re storing and how it’s being used. Some people care.

Some themes—Avada for instance—include a cookie law feature that you can enable. But there are also plugins. Here are three:

  • Complianz – GDPR/CCPA Cookie Consent

  • GDPR Cookie Compliance (CCPA, DSGVO, Cookie Consent)

  • Cookie Notice & Compliance for GDPR/CCPA

5.8 Ensuring accessibility

The CDC says one in four Americans experiences some type of disability. It’s wrong to assume that website-related disability refers only to people challenged by full blindness. Other conditions that may challenge your visitors include partial vision, auditory problems, color blindness, cognitive challenges, and neurological or sensory problems. For example, people with cognitive impairments such as dementia, autism, or traumatic brain injury may have trouble dealing with layout, design, and navigability. And a crowded layout or small buttons and links can cause difficulties for people with limited use of their hands.

So, a large portion of your audience likely will appreciate you taking their needs into account. And there’s nothing to prevent you from ensuring that your site is accessible to all visitors. Here are some things to check or put in place:

  • Does your typeface color strongly contrast against the background for ease of readability? Visit the WebAIM site for a useful Contrast Checker test: https://webaim.org/resources/contrastchecker/.

  • Is your typeface reasonably large (again, for the sake of readability)?

  • Add alternative text to all your images. Here’s how: in your Media Library, click an image to open its Edit Image screen. Scroll down and describe the image in the Alternative Text field. This description is read aloud to people with visual challenges. It won’t be visible to other visitors unless for some reason the image itself can’t be displayed.

  • When you embed a YouTube video, add captions to it for the hearing impaired. (When adding video to your site, it’s best to upload the video to YouTube and then embed the video in one of your site’s pages. This technique is explained in an upcoming chapter.)

How to add captions to an embedded video

Because it’s not easy to automate site accessibility, you’ll find only a few plugins that attempt the job. And some of these plugins have even raised legal problems (not fixing all the problems or inadequate compliance, thereby potentially exposing you to legal challenges). So, manually ensuring accessibility is recommended. For one thing, you can tweak your site by hand fairly quickly to fix any bad typography problems. And remember to fill in the Alt Text field when adding images to your pages.

One helpful tool for making your site available to all is an accessibility checker. Here are three for you to consider:

  • The WAVE accessibility checker (https://webaim.org) is a free utility that checks web pages for accessibility. It gives you useful alerts about problems that need attention.

  • Monsido Accessibility Checker (https://monsido.com/platform/web-accessibility) is a paid system that scans web pages for accessibility problems. It provides a detailed report and recommendations on how to fix any problems.

  • Siteimprove Accessibility Checker (https://mng.bz/6YWR) is another paid tool that checks web pages. You get a detailed report on problems and ways to fix them. It also includes a nice feature that simulates how your website looks to someone with visual impairment.

5.9 Adding a sidebar

Widgets are similar to plugins, but are usually smaller and offer more limited functionality. Widgets are placed into a site’s sidebar, which is a usually narrow strip of content containing one or more widgets (such as a small text box displaying a store address, hours, and phone number).

A sidebar is an important element of many sites. Like the menu, a sidebar appears on each page unless you turn it off for certain pages, usually because it would be redundant. You can use the Widget Visibility module in Jetpack to select which pages will display which widgets.

On a sidebar, you put content that deserves to be visible most everywhere within your site. A sidebar can contain various kinds of content. Figure 5.13 illustrates a restaurant website’s home page, with featured menu items on a right-side sidebar.

Figure 5.13 A sidebar can contain specialized content, such as this restaurant’s currently featured menu items.

A sidebar often contains subsidiary content stacked in a narrow, usually vertical, column. A sidebar takes up perhaps only a fifth the width of the primary content. Its narrowness is why it contains widgets with their more limited functionality when compared to plugins.

So, it’s a thin strip of real estate on which site designers can display a clickable map that expands to display your store location, a clickable events calendar, or perhaps a couple of sample blog posts or testimonials.

Your theme determines how many types and locations of sidebars are available. Look in your Appearance > Widgets screen. They can be on the left, right, or below the main content. Also, in some cases, a sidebar displays the site menu.

Let’s try adding a widget to a sidebar using WordPress’s Twenty Sixteen theme. Follow these steps:

   1.  Go to your Plugins > Add New screen and search for the Classic Widgets plugin (the one with 2 million installations). Install and activate it. Without this plugin, you have to use the Gutenberg version of the sidebar management system.

   2.  Go to the Appearance > Widgets screen. Displayed there is a list of default widgets and some Jetpack widgets as well, as shown in figure 5.14.

Figure 5.14 Some widgets come preinstalled by your theme, and others ride in on Jetpack during its installation.

TIP If you see some widgets already installed (named Block) in the sidebar field, just open them (by clicking the small triangle icon in their title bar) and delete them from the sidebar field.

   3.  Scroll down and locate the Text widget. Then, drag and drop it in the Sidebar field in the upper-right corner.

   4.  It automatically opens and displays a text box similar to the text box in your Edit Page screen. You can add a title and some text. Then, click the Save button to store your sidebar.

Tip Notice in figure 5.14 the Visibility button in the Text widget’s field. Click that button if you don’t want this widget visible in certain pages. If you don’t see a Visibility button, go to your Jetpack > Modules screen, and activate the Widget Visibility module.

   5.  Click your site title in the upper-left corner so you can view your site’s frontend and check out the new sidebar you just created.

Play around with some of the other widgets as well. There are some useful ones already available in the Widgets screen. And you can find many more by going to your Plugins > Add New screen and searching for widgets.

5.10 Creating your own plugins and widgets with AI

Creating plugins yourself sounds like a heavy-duty programming job, and it certainly can be. But as is so often the case, AI can manage what most of us can’t.

As a demonstration, we’ll ask GPT to create a WordPress widget that displays the current date and time. After that, we’ll ask AI to generate some HTML, CSS, and JavaScript code for us.

Warning Be sure to back up your site before testing custom plugins or widgets.

Tip Recall that some researchers have found that the best way to start a prompt is to use the phrase: “Take a deep breath and work step-by-step.”

5.10.1 AI generates a widget

To create a widget, I used this prompt with GPT:

Take a deep breath and work step-by-step. Please create a WordPress plugin that displays the current time (Greenwich Mean Time) and date in a text widget on the sidebar. Make the plugin into a text file of PHP code that I can make into an uploadable .zip file. Make the background of the widget light blue. And append “Greenwich Mean Time”.

GPT responded with this code:

 
<?php
/*
Plugin Name: GMT Time and Date Widget
Description: A widget that displays the
current Greenwich Mean Time (GMT) and date.
Version: 1.0
Author: ChatGPT
*/
 
// Register and load the widget
function gmt_load_widget() {
    register_widget('gmt_widget');
}
add_action('widgets_init', 'gmt_load_widget');
 
// Creating the widget 
class gmt_widget extends WP_Widget {
 
    function __construct() {
        parent::__construct(
            // Base ID of your widget
            'gmt_widget', 
 
            // Widget name will appear in UI
            __('GMT Time and Date Widget', 'gmt_widget_domain'), 
 
            // Widget description
            array('description' => __('Displays the
current Greenwich Mean Time (GMT) and date.',
'gmt_widget_domain'))
        );
    }
 
    // Creating widget frontend
    public function widget($args, $instance) {
        echo $args['before_widget'];
        
        // This is where you run the code and display the output
        $current_time = gmdate("H:i:s");
       $current_date = gmdate(«Y-m-d»);
        echo "<div style='background-color: lightblue; padding: 10px;'>";
        echo "<strong>Time:</strong> " . $current_time . "<br>";
        echo "<strong>Date:</strong> " . $current_date;
        echo "<p>Greenwich mean time</p>";
        echo "</div>";
        
        echo $args['after_widget'];
    }
}
?>

To try this on your site, back up the site, and then follow these steps:

  1. Copy the PHP code that GPT generated.

  2. Paste the code in Notepad.

  3. Save the text file named TimeandDate.txt.

  4. Right-click that file, and rename it TimeandDate.php.

  5. Right-click the file again, and choose Compress to Zip file.

  6. Go to your Plugins > Add New screen.

  7. Click Upload Plugin, and then click Choose File.

  8. Upload your zip file.

  9. Click Install Now.

  10. Click Activate Plugin.

  11. Now you need to add the widget to your sidebar, so go to the Appearance > Widgets screen.

  12. Locate the GMT Time and Date Widget. (WordPress fetches the name of the widget from the code AI generated.)

  13. Open its module by clicking the black triangle icon.

  14. Click Add Widget.

Now check it out. Click your site’s title in the upper left of the WordPress backend to view the frontend. Scroll down to the bottom of the sidebar to view the widget. Each time the page is refreshed, the time will refresh.

Tip Currently, the plugins and widgets AI creates are a hit-and-miss proposition. More complex plugins cause AI to struggle. And that can require your intervention and debugging. Output is beyond AI right now (such as emailing to you the results of a form your visitors fill out.) But given AI’s speed of self-improvement, give it a few months—who knows how much more sophisticated its plugins will become?

Understanding website languages

The language of HTML primarily focuses on design (describing how the web page looks—mostly in terms of its structure, its layout). And the CSS language works with HTML, refining the HTML design (but also including special effects such as transitions and animation). JavaScript is a language that describes behavior. For example, you use <script> to tell the browser what steps to take when a button is clicked. JavaScript (like HTML) usually runs in the browser, so it can do things without needing to refresh the page. Like displaying an extra block of text. As you saw in the previous example code, you can insert JavaScript code within HTML using the <script> tag.

Another website language, PHP, also frequently governs behavior, but it runs server-side (in the cloud on your hosting service’s drives). Several other programing languages can also be employed server-side to control website behaviors.

Summary

  • Plugins add functionality to a website—adding tools and features useful to both the site creator and the audience. And some plugins are valuable in most every site: for security, Wordfence and UpdraftPlus; for adding content, Easy Google Fonts, and Advanced Editor Tools; and several other categories.

  • You need to pay attention to a plugin’s popularity and version. If it has few installations or isn’t being kept up-to-date by its developers, avoid it.

  • Installing too many plugins increases the possibility of side effects and conflicts between plugins. Too many can also slow down your page-load time, which can annoy your visitors with slower internet speeds.

  • Seriously consider installing the 46 modules offered in the Jetpack set of widgets and plugins. Although often lightweight, many of them are quite functional.

  • Your site should be inviting to people with special needs, so take the time to modify your pages to make them easily accessible to people with hearing, vision, or other challenges. Pay particular attention to typefaces and providing Alt Text descriptions of your images.

  • Sidebars offer several benefits to a website: they can appear on each page; contain small data and media, such as business hours or a map to your store; and display widgets appropriate to the content of individual pages.

  • With AI assisting (actually it does all the work), you can create custom plugins and widgets for your site, or for sale. You can also ask AI to write complete HTML, CSS, JavaScript, and PHP executable code.
